## 1. 概述与产品关系

### 1.1 三家厂商产品体系总览

AK1和AK2是两代超声波泊车辅助（APA）系统方案，核心区别在于所采用的**传感器芯片架构**不同。目前市场上涉及三家主要芯片厂商：**Elmos（德国）**、**极海半导体（中国）** 和 **佑航科技（中国）**，各自提供的产品覆盖AK1或AK2系统。

| 厂商 | 所在地 | AK1系统产品<br>（定频/IO总线/硬件固化） | AK2系统产品<br>（Chirp并行/DSI3总线/软件可编程） | 产品线覆盖 |
|------|------|-----------------------------------|--------------------------------------------|----------|
| **Elmos** | 德国 | **E524.09**（IO 3线）<br>**E524.08**（2线供电调制）<br>**E524.11~E524.24**（LIN） | **E524.17 / E524.20**（DSI3，ARM Cortex-M0+DSP，探头主芯片）<br>**E521.42**（SPI转双DSI3总线主收发器）<br>**E524.32 / E524.33**（新一代，内置高压驱动，无需变压器） | **AK1 + AK2 全覆盖** |
| **极海半导体** | 中国（西安） | **GURC01**（超声波测距传感器V1.0，IO，~2023量产） | **G32A217**（DSI3，ARM Cortex-M0，ASIL-B，量产）<br>**G32A221**（V3.0，变频Chirp 30-125kHz，DSI3，ASIL-B，即将上市）<br>**GAIF2042**（双通道DSI3收发器V1.0，444kbit/s，ASIL-B）<br>**GAIF2044**（双通道DSI3收发器V2.0，666kbit/s，ASIL-B） | **AK1 + AK2 全覆盖** |
| **佑航科技** | 中国（珠海） | **DJ628.19**（对标E524.09，IO/ESI） | **DJ628.30**（对标E524.20，ESI/DSI3，RISC-V）<br>**DJ628.27**（对标E524.17，含微碰撞/OTA/哨兵） | **AK1 + AK2 全覆盖** |

### 1.2 AK1系统：Elmos E524.09 vs 佑航 DJ628.19

AK1系统以**Elmos E524.09**为原型芯片，**佑航 DJ628.19** 为其国产替代方案，极海半导体有独立自研AK1产品 **GURC01**（超声波测距传感器V1.0，~2023年量产）但不与Elmos/佑航互为对标。

| 对比维度 | Elmos E524.09 | 佑航 DJ628.19 | 差异说明 |
|---------|--------------|--------------|---------|
| **芯片定位** | AK1原版（2代超声波驱动器+硬件处理器） | AK1国产替代 | DJ628.19对标E524.09，功能基本兼容 |
| **处理器** | 无MCU（纯硬件信号处理） | 硬件信号处理器（同类架构） | 两者均无用户可编程MCU |
| **通信接口** | IO（3线）/ LIN | IO / LIN / **ESI可选** | 佑航增加ESI接口选项 |
| **通信速率** | ~6.7kbit/s（IO） | 同IO速率，ESI更高 | ESI模式速率更高 |
| **发波模式** | 仅定频 | 仅定频 | 两者相同，均不支持Chirp |
| **ADC** | 内部使用 | **16bit Sigma-Delta ADC** | 佑航ADC精度更高 |
| **MTP存储** | EEPROM（校准参数） | **48KB MTP** | 佑航存储更大，可存储更多配置 |
| **SRAM** | 无 | **24KB** | 佑航有SRAM |
| **AATG自适应阈值** | ✗ | ✗ | 两者均不支持 |
| **ISO26262** | ✗ | ✗ | 两者均无功能安全认证 |
| **封装** | QFN20（5×5mm） | QFN20 | 封装兼容 |
| **供货地区** | 德国，进口 | 中国大陆，本土化 | 佑航可本土化供应链 |

> **结论**：DJ628.19是E524.09的直接替代，引脚封装兼容，核心功能一致，在ADC精度和存储容量上有一定提升，同时增加ESI接口选项，适合AK1系统国产化替换。

### 1.3 AK2系统：Elmos E524.17/E524.20 vs 极海 G32A217 vs 佑航 DJ628.30 / DJ628.27

AK2系统三家厂商均有产品。**Elmos E524.17/E524.20** 是Elmos原版AK2探头主芯片（配合 **E521.42** DSI3总线主收发器组网），**极海 G32A217** 为国内代表，**佑航 DJ628.30/DJ628.27** 为国产替代与增强方案。

| 对比维度 | Elmos E524.17/E524.20 | 极海 G32A217 | 佑航 DJ628.30 | 佑航 DJ628.27（增强） | 差异说明 |
|---------|----------------------|------------|--------------|-------------------|---------|
| **芯片定位** | AK2原版探头主芯片 | AK2国内基准 | AK2国产替代 | AK2国产增强版 | DJ628.27在30基础上增加独家功能 |
| **处理器** | ARM Cortex-M0 + DSP | ARM Cortex-M0（32位） | **32位RISC-V** | **RISC-V + DSP** | Elmos/佑航27含DSP，G32A217无独立DSP |
| **通信接口** | DSI3（行业标准） | DSI3（行业标准） | **ESI / DSI3双模** | ESI / DSI3双模 | 佑航支持ESI更高速，兼容DSI3 |
| **通信速率** | 444kbit/s（DSI3） | 444kbit/s（DSI3） | **888kbit/s（ESI）** | 888kbit/s（ESI） | 佑航ESI速率是DSI3的2倍 |
| **系统组网芯片** | **E521.42**（SPI转双DSI3主收发器） | ECU直接集成DSI3主机 | ECU直接集成 | ECU直接集成 | Elmos方案需额外E521.42芯片 |
| **发波模式** | 定频 + Chirp | 定频 + 线性Chirp | 定频 + Chirp + **MSK + FSK** | 定频 + Chirp + FSK | 佑航支持更多调制方式 |
| **直接驱动** | **E524.32/33内置高压驱动**（新一代，无需变压器） | 需外部电路 | 需外部电路 | 需外部电路 | Elmos新一代E524.32/33可无变压器直驱压电陶瓷 |
| **ADC** | 高精度（DSP辅助） | 10位ADC × 1 | **16bit Sigma-Delta** | **16bit Sigma-Delta** | 佑航ADC位数更高 |
| **MTP存储** | 内置Flash | 32KB MTP | **48KB MTP** | **48KB MTP** | 佑航存储更大 |
| **SRAM** | 内置SRAM | 4KB | **24KB** | **24KB** | 佑航SRAM更大 |
| **AATG自适应阈值** | ✓ | ✓ | ✓ | ✓ | 四者均支持 |
| **Chirp并行测量** | ✓（12探头并行） | ✓ | ✓ | ✓ | 四者均支持 |
| **完整包络输出** | ✓（含原始数据） | ✓ | ✓ | ✓ | 四者均支持 |
| **ISO26262** | **ASIL-B** | **ASIL-B** | **ASIL-B** | **ASIL-B** | 四者均通过ASIL-B |
| **微碰撞检测** | ✗ | ✗ | ✗ | **✓（独有功能）** | DJ628.27独家弹性波微碰撞检测 |
| **OTA固件升级** | ✗ | ✗ | ✗ | **✓** | DJ628.27支持OTA |
| **哨兵模式** | ✗ | ✗ | ✗ | **✓** | DJ628.27停车后静默守卫 |
| **封装** | QFN | QFN20 | QFN20 | QFN20 | — |
| **供货地区** | 德国，进口 | 中国（西安） | 中国（珠海） | 中国（珠海） | 国内两家均可本土化供应 |

> **结论**：Elmos E524.17/E524.20是AK2系统的原版芯片，性能标杆，但需额外E521.42组网，且为进口产品；极海G32A217是国内自研AK2方案，不与Elmos/佑航互为对标，但具备完整AK2特征（DSI3+Chirp+ARM M0+ASIL-B）；佑航DJ628.30**对标E524.20**，在通信速率（888kbit/s ESI）、ADC精度（16bit）和存储（SRAM 24KB）上全面超越E524.20；佑航DJ628.27**对标E524.17**，在E524.17功能基础上新增微碰撞检测、OTA升级、哨兵模式三项独家功能，面向L2+自动驾驶。

### 1.5 分析视角

APA系统包含 **8~12个超声波传感器节点** 和 **1个APA ECU**，完整系统由两层组成：

```
┌──────────────────────────────────────────────────────────┐
│                   完整APA系统组成                          │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │              APA ECU（主控制器）                   │   │
│  │  · 多路传感器数据汇聚与融合                         │   │
│  │  · 障碍物建图（Occupancy Grid）                    │   │
│  │  · 车位检测与路径规划                               │   │
│  │  · 泊车轨迹跟踪控制                                │   │
│  └──────────────────────────────────────────────────┘   │
│                     ↑ 通信总线 ↑                          │
│  ┌──────┐ ┌──────┐ ┌──────┐  ...  ┌──────┐ ┌──────┐   │
│  │传感器1│ │传感器2│ │传感器3│       │传感器11│ │传感器12│  │
│  │(芯片)│ │(芯片)│ │(芯片)│       │(芯片) │ │(芯片) │  │
│  └──────┘ └──────┘ └──────┘       └──────┘ └──────┘   │
│                                                          │
│  AK1: 每个节点用 E524.09（Elmos）或 DJ628.19（佑航）             │
│  AK2: 每个节点用 E524.17/20（Elmos）、G32A217（极海）或 DJ628.30/27（佑航）│
└──────────────────────────────────────────────────────────┘
```

**AK1与AK2的差异，不仅体现在单颗传感器芯片能力上，更体现在整个系统的拓扑结构、总线带宽、测量并发性、ECU侧算法输入质量等全局层面。**

## 2. 系统架构差异

### 2.1 AK1系统整体架构（基于E524.09）

E524.09是Elmos第二代超声波驱动器与信号处理器，**内部无MCU**，算法完全硬件固化，通过IO（3线）或LIN与ECU通信。系统必须串行轮询各传感器。

```
┌────────────────────────────────────────────────────────────────────┐
│                AK1 完整APA系统架构（典型12传感器）                    │
│                                                                    │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │                         APA ECU                               │ │
│  │  ┌──────────────┐   ┌─────────────────┐   ┌──────────────┐  │ │
│  │  │ IO/LIN主机   │   │  数据融合算法    │   │  APA应用层   │  │ │
│  │  │ 控制器×12    │   │  ·障碍物建图    │   │  ·车位检测   │  │ │
│  │  │ (逐路轮询)   │   │  ·点云拼接      │   │  ·路径规划   │  │ │
│  │  └──────┬───────┘   └─────────────────┘   └──────────────┘  │ │
│  └─────────┼────────────────────────────────────────────────────┘ │
│            │  12路独立IO/LIN线（每传感器1根）                       │
│  ─────────────────────────────────────────────────────────────────│
│  前保: IO线1→[E524.09_S1]  IO线2→[E524.09_S2]  IO线3→[S3]  ...  │
│  侧方: IO线5→[E524.09_S5]  IO线6→[S6]  IO线7→[S7]  IO线8→[S8]  │
│  后保: IO线9→[E524.09_S9]  IO线10→[S10]  IO线11→[S11]  ...      │
│                                                                    │
│  E524.09每个节点特征：                                              │
│  · 纯硬件信号处理器（无MCU）                                        │
│  · 上报内容：回波时间（距离）+ 峰值                                  │
│  · 仅支持定频发波                                                   │
│  · IO 3线通信，速率低                                               │
│                                                                    │
│  测量方式：串行轮询，12路顺序激活                                     │
│  测量周期：~600ms（12路×50ms/路，估算）                              │
└────────────────────────────────────────────────────────────────────┘
```

### 2.2 AK2系统整体架构（基于G32A217）

G32A217集成32位ARM Cortex-M0，通过DSI3总线与ECU通信，支持多节点总线模式。配合Chirp调制，可实现多路并行测量。

```
┌────────────────────────────────────────────────────────────────────┐
│                AK2 完整APA系统架构（典型12传感器）                    │
│                                                                    │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │                         APA ECU                               │ │
│  │  ┌──────────────┐   ┌─────────────────────┐  ┌───────────┐  │ │
│  │  │ DSI3主机控制  │   │   增强数据融合算法    │  │ APA应用层 │  │ │
│  │  │ ·总线仲裁    │   │  ·包络数据分析       │  │ ·高精车位 │  │ │
│  │  │ ·自动寻址    │   │  ·置信度加权融合     │  │ ·路径规划 │  │ │
│  │  │ ·链路诊断    │   │  ·Chirp并行调度      │  │ ·轨迹控制 │  │ │
│  │  └──────┬───────┘   └─────────────────────┘  └───────────┘  │ │
│  └─────────┼────────────────────────────────────────────────────┘ │
│            │  1~2条DSI3总线（多从机共享）                           │
│  ─────────────────────────────────────────────────────────────────│
│  DSI3总线_前: ─[G32A217_S1(0x01)]─[S2(0x02)]─[S3(0x03)]─...     │
│  DSI3总线_后: ─[G32A217_S7(0x07)]─[S8(0x08)]─[S9(0x09)]─...     │
│                                                                    │
│  G32A217每个节点特征：                                              │
│  · 32位ARM Cortex-M0（完全可编程）                                  │
│  · 上报：距离+回波信息+包络数据+诊断                                 │
│  · 支持定频+线性Chirp调频                                           │
│  · DSI3总线，最高444kbit/s                                          │
│                                                                    │
│  测量方式：Chirp并行，前/后6路分组同时测量                            │
│  测量周期：~100ms（2组×50ms，估算），刷新率提升约6倍                  │
└────────────────────────────────────────────────────────────────────┘
```

### 2.3 系统架构核心差异

| 对比维度 | AK1系统（E524.09） | AK2系统（G32A217） |
|---------|-----------------|-----------------|
| **系统拓扑** | 星型（每传感器独立IO线） | 总线型（多节点共享DSI3） |
| **12路布线** | **12根IO/LIN线** | **2根DSI3总线** |
| **测量模式** | **串行轮询** | **Chirp并行** |
| **系统刷新率** | ~1.7Hz（600ms/轮） | **~10Hz（100ms/轮）** |
| **传感器算法** | 硬件固化，仅上报距离+峰值 | 软件可编程，上报丰富数据 |
| **ECU算法负担** | 重（算法全在ECU） | 轻（节点侧分担预处理） |
| **感知维度** | 距离（超声波单模） | 距离+包络+类型（超声波） |

## 3. 传感器芯片硬件差异

### 3.1 芯片架构对比

| 参数 | AK1（Elmos E524.09） | AK2（极海G32A217） |
|------|---------------------|-----------------|
| **芯片定位** | 第二代超声波驱动器+信号处理器 | 超声波传感和信号处理SOC |
| **处理器** | **无MCU**（纯硬件信号处理） | **32位ARM Cortex-M0** |
| **处理器时钟** | N/A（12MHz内部振荡器用于IO通信和信号处理） | 内部时钟（可编程分频） |
| **算法实现** | 硬件固化电路 | 软件（嵌入式C程序） |
| **可编程性** | 仅EEPROM参数配置 | 完全可编程 |
| **调试接口** | JTAG（5引脚） | **SWD** |

### 3.2 存储配置

| 参数 | AK1（E524.09） | AK2（G32A217） |
|------|--------------|---------------|
| **程序存储** | 无（硬件固化） | **MTP：32KB** |
| **SRAM** | 无 | **SRAM：4KB** |
| **EEPROM** | 少量（校准参数存储） | **EEPROM：256B** |
| **用途** | EEPROM仅存校准值（振荡器频率、驱动电流、增益等） | MTP存储嵌入式固件，EEPROM存标定参数 |

> **关键差异**：E524.09没有用户程序存储器，算法完全由Elmos固化在芯片内，用户只能通过EEPROM配置有限参数；G32A217内置32KB MTP存储用户固件，算法可自主开发和升级。

### 3.3 驱动能力对比

| 参数 | AK1（E524.09） | AK2（G32A217） |
|------|--------------|---------------|

## 4. 通信接口与总线差异

### 4.1 系统总线架构对比（最重要的系统级差异）

#### AK1 IO/LIN总线方案（E524.09）

E524.09支持2线（E524.08，供电线调制）或3线（E524.09，独立IO线）通信。AK1通常使用3线IO模式：

```
APA ECU
├── IO控制器通道 × 12（每路独立）
│    ├── CH1 线(GND + VSUP + IO) ─── [E524.09_S1]   前左1
│    ├── CH2 ─── [E524.09_S2]   前左2
│    ├── CH3 ─── [E524.09_S3]   前右1
│    ├── CH4 ─── [E524.09_S4]   前右2
│    ├── CH5 ─── [E524.09_S5]   侧左1
│    ├── CH6 ─── [E524.09_S6]   侧左2
│    ├── CH7 ─── [E524.09_S7]   侧右1
│    ├── CH8 ─── [E524.09_S8]   侧右2
│    ├── CH9 ─── [E524.09_S9]   后左1
│    ├── CH10 ─── [E524.09_S10] 后左2
│    ├── CH11 ─── [E524.09_S11] 后右1
│    └── CH12 ─── [E524.09_S12] 后右2

IO通信时序（T_BIT=150μs/bit，~6.7kbit/s）：
  · SEND命令：T_SND ≈ 100μs低电平
  · RECEIVE命令：T_REC ≈ 150μs低电平
  · 数据位：T_BIT0=100μs，T_BIT1=50μs
  · 传感器应答：电流调制输出
```

#### AK2 DSI3总线方案（G32A217）

G32A217原生支持DSI3协议，支持菊链（DSII→DSIO）总线模式：

```
APA ECU
├── DSI3主机控制器 × 1~2
│    │
│    ├── DSI3总线_前 ─[G32A217_S1]─[S2]─[S3]─[S4]─[S5]─[S6]
│    │               (DSII链接到ECU，DSIO链接下一节点)
│    │               前保4路 + 侧方2路，地址0x01~0x06
│    │
│    └── DSI3总线_后 ─[G32A217_S7]─[S8]─[S9]─[S10]─[S11]─[S12]
│                    侧方2路 + 后保4路，地址0x07~0x0C

DSI3通信特性：
  · 最高444kbit/s（CRM模式，与Elmos/onsemi DSI3兼容）
  · 主机发电压命令，从机电流应答
  · 自动寻址（上电DM模式自动分配地址）
  · CRC错误检测
  · 支持点对点和菊链总线两种拓扑
```

### 4.2 通信接口参数对比

| 参数 | AK1（E524.09 IO模式） | AK2（G32A217 DSI3模式） |
|------|---------------------|----------------------|
| **协议标准** | Elmos专有IO协议 | DSI3（SENT/电流调制，行业标准） |
| **通信速率** | ~6.7kbit/s（@T_BIT=150μs） | **最高444kbit/s** |
| **速率倍数** | 基准 | **约66倍** |
| **总线拓扑** | 点对点（每传感器独立） | 总线/菊链（多节点共享） |
| **自动寻址** | ✗（需人工配置） | ✓（DSI3 DM模式自动寻址） |
| **12路布线数** | **12根IO线** | **2根DSI3总线** |
| **ECU接口** | 12路IO控制器 | 1~2路DSI3主机 |
| **错误检测** | ✗（无CRC） | ✓（DSI3 CRC校验） |
| **链路诊断** | ✗ | ✓ |

### 4.3 系统带宽与数据能力

| 指标 | AK1（E524.09） | AK2（G32A217） |
|------|--------------|---------------|
| **可用总线带宽** | ~6.7kbit/s/路（12路×6.7=约80kbit/s总计，仍串行） | **444kbit/s**（总线共享，高效复用） |
| **单次测量上报数据** | 回波时间（距离）+ 有限状态 | 距离+回波强度+包络数据+诊断信息 |
| **包络数据传输** | 仅通过IO模拟电流输出（6bit DAC，需额外时间） | ✓（高速DSI3可传输完整包络，供ECU做AI处理） |
| **多传感器并行** | ✗（串行） | ✓（Chirp编码隔离，并行采集） |

### 4.4 E524.09 IO通信时序详解

```
AK1 IO时序（E524.09，以SEND命令为例）：

ECU发SEND命令：
[高]──────────────────[低，T_SND≈100μs]──────────────[高，等待回波]

传感器回波应答（电流调制）：
[高]────[低T_BIT0≈100μs/T_BIT1≈50μs]────[位数据串行]──────

测距完成后，ECU请求数据（READ命令）：
  →传感器上报：16bit回波时间 + 状态字节

完整一路交互时间：命令~300μs + 等待回波最长~17ms（3m测距）+ 数据读取~5ms
12路串行总耗时：约（17ms + 5ms + 3ms）× 12 ≈ 300ms/轮（3m量程）
```

## 5. 传感器节点侧信号处理差异（MCU/硬件侧）

> **本章范围**：传感器芯片内部（E524.09硬件电路 / G32A217 ARM Cortex-M0）对超声波信号的处理过程，不涉及ECU侧数据融合算法。

### 5.1 信号处理架构概览

两代系统的传感器节点侧处理架构有根本性的不同：

| 架构特征 | AK1节点（E524.09） | AK2节点（G32A217） |
|---------|-----------------|-----------------|
| **处理方式** | 全硬件固化电路 | 软硬件协同（硬件模块 + ARM Cortex-M0软件） |
| **用户可编程** | 仅EEPROM参数（出厂前配置） | 完全可编程（运行时动态修改） |
| **算法定制** | 不可修改 | 用户C代码完全自定义 |
| **输出给ECU** | 距离+峰值（简单） | 距离+强度+包络+诊断（丰富） |

**AK1 节点侧信号处理链（E524.09，全硬件，不可更改）：**

```
                     ┌── AK1 节点侧（E524.09 硬件固化）──────────────────────────┐
换能器回波 ──→ 差分放大（TIA）──→ BPF带通滤波 ──→ 数字放大器（DAMP，127步）
                     │
                     ├──→ STC 灵敏度时间控制（时变增益，EEPROM配置）
                     ├──→ ATG 自动阈值生成（固化算法）
                     ├──→ STG 静态阈值生成（EEPROM配置）
                     ├──→ NFTG 近场阈值生成（固化）
                     ├──→ FTC 快时间常数算法（固化）
                     └──→ EPD 回波峰值检测（固化）
                                │
                                ▼
                     IO上报：回波时间（距离）+ 峰值状态   ──→ 送ECU
                     （包络仅可通过IO模拟6bit DAC读出）
└──────────────────────────────────────────────────────────────────────────────┘
```

**AK2 节点侧信号处理链（G32A217，软硬件协同）：**

```
                ┌── AK2 节点侧（G32A217 软硬件协同）──────────────────────────────┐
换能器回波 ──→ ASP模拟信号处理（硬件）──→ 10位ADC采样
                │
                ▼  ┌─ 硬件DSP模块（固化，参数可编程）─────────────────┐
                   │ · BPF包络处理器（多种滤波器，带宽软件配置）        │
                   │ · DAMP数字放大 / DGC动态数字增益（软件配置）       │
                   │ · FTC 快时间常数（软件配置）                       │
                   │ · STG 静态阈值 / ATG 自动阈值（软件配置）          │
                   │ · AATG 高级自适应阈值（硬件+软件协作）             │
                   │ · NFD 近场检测（专用硬件模块）                     │
                   │ · MDI包络数据接口（DMA→SRAM）                      │
                   └───────────────────────────────────────────────────┘
                                │         │
                    回波时间/强度（硬件）   包络数字数据（DMA传到SRAM）
                                │         │
                                ▼         ▼
                   ┌─ ARM Cortex-M0 用户软件（完全可定制）──┐
                   │ · 包络形态分析（软/硬目标分类）          │
                   │ · 余振频率/时间进一步处理                │
                   │ · 自定义回波评估逻辑                     │
                   │ · 诊断结果生成                          │
                   │ · 置信度/强度信息计算                    │
                   └─────────────────────────────────────────┘
                                │
                                ▼
                   DSI3上报：距离+强度+包络+诊断信息   ──→ 送ECU
└────────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 节点侧算法模块对比（MCU/硬件侧）

> 以下算法均运行在**传感器节点内部**（E524.09硬件电路 或 G32A217片上MCU/DSP），**不涉及ECU**。

| 算法模块 | 运行位置 | AK1节点（E524.09） | AK2节点（G32A217） |
|---------|---------|-----------------|-----------------|
| **BPF带通滤波器** | 节点硬件 | ✓（固化，带宽随NPULSES自动选择） | ✓（包络处理器，带宽软件可配） |
| **STC 时变增益** | 节点硬件 | ✓（固化，EEPROM配置起始时间/斜率） | ✓（DGC动态数字增益，运行时可配） |
| **STG 静态阈值** | 节点硬件 | ✓（固化，EEPROM配置） | ✓（软件可配，运行时修改） |
| **ATG 自动阈值** | 节点硬件 | ✓（固化算法） | ✓（软件可配） |
| **AATG 高级自适应阈值** | 节点硬件+MCU | ✗（不支持） | ✓（硬件+ARM协同，运行时自适应环境） |
| **FTC 快时间常数** | 节点硬件 | ✓（固化） | ✓（软件可配） |
| **NFTG 近场阈值** | 节点硬件 | ✓（固化） | 由NFD模块覆盖 |
| **NFD 近场检测** | 节点硬件 | 有限（NFTG辅助实现） | ✓（专用NFD硬件模块） |
| **EPD 回波峰值检测** | 节点硬件 | ✓（固化） | ✓（硬件完成，MCU可进一步访问） |
| **DAMP 数字放大器** | 节点硬件 | ✓（127步，0~47.89dB） | ✓（DGC，步长更灵活） |
| **余振频率/时间测量** | 节点硬件+MCU | 有限（基础测量） | ✓（专用硬件测量，ARM进一步处理） |
| **包络数据采集(MDI)** | 节点硬件+MCU | 有限（IO模拟6bit DAC输出） | ✓（DMA传输完整数字包络到SRAM） |
| **Chirp线性调频发波** | 节点硬件 | ✗（仅定频发波） | ✓（PUG模块支持，软件配置） |
| **Chirp匹配接收滤波** | 节点DSP | ✗ | ✓（配合Chirp发波的解调滤波） |
| **包络形态/目标分类** | 节点MCU（软件） | ✗（无MCU） | ✓（ARM用户软件自定义实现） |
| **置信度计算** | 节点MCU（软件） | ✗ | ✓（ARM用户软件计算，上报ECU） |

### 5.3 节点侧阈值算法详细对比

| 特性 | AK1节点（E524.09） | AK2节点（G32A217） |
|------|-----------------|-----------------|
| **阈值算法类型** | STG（静态）/ ATG（自动）/ NFTG（近场） | STG / ATG / **AATG（高级自适应）** |
| **AATG自适应** | ✗（无） | ✓（运行时动态感知环境噪声，自动调整阈值曲线） |
| **阈值配置方式** | EEPROM预存（出厂前标定，上电后不可改） | 寄存器配置（ARM可运行时修改，适应不同场景） |
| **阈值曲线套数** | A/B/C三套（出厂前选定） | A/B两套（ARM运行时可扩展） |
| **地库场景适应** | 差（固化阈值遇多路径强回波误报率高） | 好（AATG自动适应，误报率低） |

### 5.4 多传感器并行发波能力（节点侧发波模式）

| 能力 | AK1节点（E524.09） | AK2节点（G32A217） |
|------|-----------------|-----------------|
| **发波调制** | **仅定频**（单一固定频率） | **定频 + 线性Chirp调频** |
| **Chirp参数** | N/A | fL、fH、步长、CODE方向均软件配置 |
| **多路并行可行性** | ✗（定频无法区分来源，只能串行） | ✓（Chirp编码区分传感器，支持并行） |
| **对系统刷新率的影响** | 12路串行，周期约600ms（~1.7Hz） | 分2组并行，周期约100ms（~10Hz） |

AK1 串行测量时序（节点侧无Chirp，只能逐路激活）：

         ↑每路约50ms                                                               总约600ms

AK2 并行测量时序（节点侧Chirp编码，ECU协调并行）：
时间轴 ─→
[S1(fH→fL)+S2(fL→fH)+S3(fH→fL)+S4+S5+S6 同时发/收]──[S7~S12同时发/收]──
                  ↑第一组约50ms                              ↑第二组约50ms
                  总约100ms（刷新率~10Hz）
```

## 6. 算法与软件差异（MCU侧 vs ECU侧分层说明）

> **本章范围**：明确区分运行在**传感器节点MCU侧**（G32A217片上ARM）的算法，以及运行在**ECU侧（APA主控）**的算法，分析两代系统在各层级算法能力和分工上的差异。

### 6.1 两层算法架构概览

APA系统的算法分布在两个层级，AK1和AK2在算法分层上存在显著差异：

```
┌─────────────────────────────────────────────────────────────────────────┐
│              APA系统算法分层（AK1 vs AK2）                               │
│                                                                         │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │                   ECU侧算法（APA主控制器）                         │  │
│  │                                                                   │  │
│  │  AK1 ECU侧（承担约80%算法负担）：                                  │  │
│  │  · 原始距离数据汇聚（仅距离+峰值，数据维度低）                       │  │
│  │  · 多路点云拼接（串行数据，时间戳对齐简单但延迟大）                   │  │
│  │  · 障碍物检测与分类（信息量不足，精度受限）                          │  │
│  │  · 车位识别（基于距离序列，识别精度有限）                            │  │
│  │  · 泊车路径规划 + 轨迹控制（1.7Hz刷新，控制平滑性差）               │  │
│  │                                                                   │  │
│  │  AK2 ECU侧（承担约60%算法负担，质量更高）：                         │  │
│  │  · 高维数据汇聚（距离+强度+完整包络+诊断，维度丰富）                 │  │
│  │  · Chirp并行时序调度与对齐（精确时间戳，实时性强）                   │  │
│  │  · 包络辅助建图（精细障碍物边界建模）                               │  │
│  │  · 高精障碍物分类（包络形态+余振特征+目标类型）                      │  │
│  │  · 高精车位识别 + 路径规划（10Hz刷新，控制更平滑）                   │  │
│  │  · 可选AI推理（1D-CNN等轻量网络，包络数据为输入）                    │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │              节点→ECU 上报数据对比（两层算法之间的数据边界）          │  │
│  │                                                                   │  │
│  │  ┌──────────────────────────────┐  ┌──────────────────────────┐  │  │
