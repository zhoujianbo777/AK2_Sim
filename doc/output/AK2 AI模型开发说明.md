## 1. 文档概述

### 1.1 目的与范围

本文档为 AK2 超声波感知系统所需的全部机器学习模型提供完整的**开发规范**，涵盖从原始传感器数据采集到模型上车部署的全生命周期工程规程。适用芯片为**佑航 DJ628.30**（ESI 总线）和**极海 G32A217**（DSI3 总线），两款芯片共用统一的 L1 数据格式（见《AK2数据采集格式定义》V2.0）。确保团队各角色（采集工程师、标注工程师、算法工程师、验证工程师、集成工程师）遵循统一的质量标准和交付物格式，实现模型的可复现性、可溯源性和持续迭代能力。

### 1.2 适用的模型列表

| 模型编号 | 模型名称 | 输入类型 | 核心任务 | 优先级 |
|---------|---------|---------|---------|--------|
| M1 | 快速特征融合 MLP（Layer-1） | EDI 特征向量（20维，逐通道） | 障碍物有效性检测 | P0（必需） |
| M2-SC | 障碍物分类 1D-CNN 骨干（Layer-2A） | 归一化包络序列（1×256） | 9类障碍物分类 + 材质评分 + 悬空高度估算 + 中间特征输出 | P0（必需） |
| M2-SA | 组内空间注意力模块（Layer-2B） | M2-SC 中间特征（N×32，N≤6路/组） | 跨通道上下文融合，精化逐路分类结果；支持 1~N 帧时序扩展 | P1（推荐） |
| M6 | 场景理解 2D-CNN（Layer-3） | 多路包络矩阵（12×256） | OGM更新 / 车位识别 | P2（可选） |

### 1.3 开发流程总览

```
数据采集 → 原始数据入库 → 数据清洗 → 数据标注 → 数据集构建
    ↓
模型训练 → 交叉验证 → 超参调优 → 最优模型选取
    ↓
float32评估 → INT8量化 → 量化精度验收 → ECU端集成验证
    ↓
输出物打包 → 代码评审 → 上线归档
```

### 1.4 模型选型依据

#### 1.4.1 超声波信号的数据特性

模型架构的选择直接来自超声波传感器信号的以下四个本质特性：

| 数据特性 | 具体描述 | 对模型设计的约束 |
|---------|---------|---------------|
| **低维结构化特征（EDI）** | 芯片内置算法对每次发射-接收提炼出20维特征（距离、幅值、置信度、回波类型、包络统计量等），维度低、语义明确、数值范围相对稳定。 | 直接接MLP，无需卷积；BatchNorm在模型内部归一化可应对传感器个体差异 |
| **1D时序结构包络（256点）** | 包络序列记录声波能量沿飞行时间轴的分布。障碍物类别信息编码在回波峰的形状（宽窄/幅值/尾波形态）和相对位置（单峰/多峰）中，是典型的局部时序模式。 | 1D-CNN参数效率比全连接高约20倍；卷积感受野随层数增长覆盖全局，MaxPool提炼平移不变特征 |
| **通道间空间相关性** | 前排和后排传感器以弧形排列，相邻传感器对同一障碍物的入射角差约5°~15°。同一障碍物必然在相邻通道产生相似但渐变的回波，单通道的分类误差可通过邻通道上下文纠正。 | 空间注意力（M2-SA）建模组内N路传感器的互参考关系，可显著降低细杆/行人的漏检率 |
| **有效通道稀疏** | 多数泊车场景下12路中有效回波的通道仅2~6路，其余为空旷/噪声状态。 | M1前置有效性筛选可使M2的实际计算量降低50%~80%，同时降低M2对空旷通道产生误报的风险 |

##### 声波信号与图像信号的稳定性差异（设计动因）

上述四个特性之外，超声波相对视觉传感器还有一个贯穿全文设计的本质差异——**单帧信噪稳定性更低、相邻帧跳变更明显**。这是 M2 采用逐通道归一化、M1 采用 BatchNorm、训练引入波形增强、以及 M2-SA 设计多帧时序融合的统一动因，需在选型阶段明确：

| 维度 | 相机（被动光强成像） | 超声波（主动声学测距） |
|------|------------------|--------------------|
| **信号获取方式** | 被动接收环境光，一帧为某时刻光强在像素上的二维投影 | 主动发射声脉冲并接收回波，测量飞行时间与回波形态 |
| **测量物理量** | 像素光强（能量） | 飞行时间 / 包络形状（链路任一环节受扰即变形） |
| **介质敏感性** | 空气对可见光近乎透明，折射率变化在车规尺度可忽略 | 声速随温度变化显著：0→30°C 约 +5%（约 331→349 m/s），直接平移包络峰位 |
| **环境扰动源** | 主要为光照突变（车灯、阳光频闪、隧道进出、自动曝光跳变） | 温度漂移、风速/气流偏折、环境声噪声（他车 USS、气动/雨滴）、多径与镜面掠射闪烁 |
| **相邻帧相关性** | 光源稳定时帧间高度相关，像素跳动小 | 同一静止目标的包络幅值与峰位仍易抖动，帧间跳变明显 |

**关键澄清（避免过度简化）**：

1. **并非"图像天生稳、声波天生抖"，而是扰动源不同**。一旦光源突变（夜间/对向车灯/阳光频闪/果冻效应），图像帧间跳变会比超声波更剧烈。二者只是各自的"坏条件"不同——图像怕光照突变，声波怕介质与声学干扰。
2. **声波跳变含可补偿的确定性成分**。温度引起的声速漂移是系统性、可建模的（温补查表 / CAN 温度信号校正，§3.1.2 已要求同步采集温度），不同于图像随机光噪；真正棘手的是随机的多径、旁瓣与镜面闪烁。

**对模型设计的约束（本文档对应处理）**：

- M2-SC 不吃绝对幅值，改吃**逐通道归一化到 [0,1] 的包络形状**（§2.1），在预处理阶段消除声速/灵敏度漂移导致的整体缩放与平移。
- M1 用 **BatchNorm**（§1.4.3）抹掉传感器个体与环境带来的量纲/幅值偏置。
- 训练数据增强专门模拟帧间跳变：幅度缩放 0.85~1.15、时间轴平移 ±5 点、高斯噪声 σ=0.01~0.03（§5.4.1）。
- M2-SA 的**多帧时序融合**（§6.4）用 T 帧加权池化平均掉随机跳变，保留稳定目标结构——若声波像图像那样帧间稳定，单帧即足够，时序融合收益将显著降低。

#### 1.4.2 分层设计逻辑

选择三层递进架构（Layer-1 / Layer-2A / Layer-2B）的核心理由是**按任务难度匹配计算代价**：

```
┌───────────────────────────────────────────────────────────────┐
│  Layer-1  M1 MLP（有效性）                                     │
│  任务难度：低  计算量：极低（~28K MACs/帧）  12路全部运行        │
│  回答问题：有没有回波？质量可信吗？                              │
└──────────────────────────┬────────────────────────────────────┘
                           │  仅对 p_valid ≥ 0.5 的通道（典型6路）
┌──────────────────────────▼────────────────────────────────────┐
│  Layer-2A  M2-SC 1D-CNN（单通道分类）                          │
│  任务难度：中  计算量：中（~752K MACs/通道）  仅有效通道运行     │
│  回答问题：这路回波是什么类型的障碍物？                          │
└──────────────────────────┬────────────────────────────────────┘
                           │  输出 feat[32]；按前/后组聚合
┌──────────────────────────▼────────────────────────────────────┐
│  Layer-2B  M2-SA（组内空间注意力，可选）                        │
│  任务难度：低  计算量：极低（~58K MACs/帧，即M2-SC的1.3%）       │
│  回答问题：相邻通道的上下文是否支持当前通道的分类结论？           │
└───────────────────────────────────────────────────────────────┘
```

每一层向上不可逆地降维（20维→1维；256维→32维→9维），避免在高维空间做复杂决策。每一层向下只传递必要信息（有效性标志 / 中间特征 feat[32]），通信开销极低。这种设计同时支持**按算力平台裁剪**：MCU仅部署M1+M2-SC（INT8），高算力平台追加M2-SA（SA~）或M6。

#### 1.4.3 各模型架构的具体选型理由

**M1：MLP（20→32→16→1）**

- **为什么不用CNN？** EDI特征是芯片内置算法的提炼结果，各维度语义明确（距离/幅值/置信度等），不存在位置相关的局部模式，卷积的空间共享假设不适用。
- **为什么先扩（20→32）再缩（32→16→1）？** 扩展层让网络有机会在高维空间发现特征间的交叉相关（如"高幅值×低置信度×宽包络"组合→地面杂波），再压缩聚合为最终判断。直接20→1会强迫过早整合，特征利用不充分。
- **为什么加BatchNorm？** 12路传感器的幅值范围因安装位置（近端/远端/角落）差异较大，BatchNorm在第一层后消除通道间的量纲偏差，使同一套权重可以共享给所有12路。

**M2-SC：1D-CNN（3卷积层 + GlobalAvgPool + 3输出头）**

- **为什么不用全连接处理包络？** 256维全连接第一层参数量至少 256×256=65K，且不具备平移不变性（同一障碍物在不同距离下回波峰位置偏移，全连接须重新学习）。1D-CNN参数约11.7K，减少5倍，且卷积核天然平移等变。
- **为什么不用RNN或Transformer？** 包络的"时间轴"对应飞行时间（非真实时序），不存在因果依赖；256点内的长程依赖较弱（回波主峰与尾波距离通常≤30点）。RNN顺序计算延迟高，Transformer显存在ECU上不友好。1D-CNN延迟比同级RNN低3~5倍。
- **为什么3层卷积使用递增通道（16→32→64）？** 浅层（kernel=7）提取波形边缘与峰形粗轮廓，中层（kernel=5）捕获峰宽和次峰关系，深层（kernel=3）识别尾波振荡；通道数递增与感受野扩大同步，保证每层特征图的信息密度基本恒定。
- **为什么设计3个输出头（分类/硬度/净高）？** 材质硬度和悬空高度与类别标签相关但不等价，分离为独立回归头可在各自监督信号下单独优化，避免任务竞争。Sigmoid确保硬度在[0,1]，ReLU确保净高非负。

**M2-SA：组内空间注意力（MultiheadAttention，4头）**

- **为什么需要跨通道信息？** 超声波分辨率约10~20cm，细杆/儿童可能仅覆盖1~2个通道导致单通道置信度低；地面杂波通常只在中心通道出现而邻近通道不出现。引入组内上下文后，模型可学习"仅1路弱回波+邻路无回波→可能是细杆"的空间规律，从而减少误报和漏报。
- **为什么按前组/后组分别处理，而非12路统一处理？** 前排和后排传感器面向相反方向，在同一帧中探测不同物理区域，跨组做注意力会将无关联的空间区域混合，反而引入噪声。
- **为什么选4头？** 4头允许模型同时从4种关系模式（近距强回波对、入射角对称对、中心-侧向差异、左右不对称）中提取上下文。N=6时注意力矩阵仅6×6，增加头数的边际收益有限，4头是参数效率与表达能力的折中。
- **为什么整体参数仅4876？** feat_dim=32已足以编码单通道的全局形状信息；注意力在32维空间操作，不需要扩维即可完成任务，额外计算开销≤0.2ms，适合作为可选层在算力受限的ECU上部署。

**M6：2D-CNN（场景理解，Layer-3）**

- **为什么用2D-CNN而非1D-CNN？** M2-SC的1D-CNN以单路256点包络为输入，输出的是"这一路传感器看到了什么"；而OGM（占据栅格地图）和车位识别需要回答"障碍物在车辆坐标系的哪个位置"。将12路包络拼为12×256矩阵后，**行维度**对应传感器的物理排布位置（空间轴），**列维度**对应飞行时间（距离轴）。同一障碍物在两个相邻传感器上同时触发会在矩阵上形成跨行的相关性条纹，2D卷积核能同时学习"哪几路传感器同时有回波"和"回波在距离轴上的形状"，1D-CNN只能逐行独立处理，无法捕捉行间的跨传感器模式。
- **为什么不直接用M2-SC的9类结果拼接后做OGM？** M2-SC的输出是每路传感器的障碍物类别概率（离散语义），丢失了回波幅值分布、多目标重叠、近端旁瓣等低级信号信息。OGM的栅格分辨率约5cm，需要对包络原始形状做精细的几何反演；用类别概率做中间层会引入不可逆的信息损失，导致OGM精度上限下降。
- **为什么输入尺寸定为12×256而非更高分辨率？** 12路传感器是硬件固定配置；256点包络对应约5m量程下的原始采样，是芯片DSI3接口的实际输出分辨率，不做降采样即可完整保留距离-幅值信息。若对应更长量程，可在预处理时截取前256点，保持输入尺寸一致。
- **为什么优先级为P2（可选）？** OGM质量强依赖于12路传感器同帧、同步的高质量数据，角落传感器的遮挡和低SNR在实际泊车场景中较常见；车位边界检测也需要密集的像素级标注，数据准备成本显著高于M1/M2的逐通道标注。APA基础功能（近距停车/障碍物减速）由M1+M2-SC即可支撑，M6在复杂多障碍或平行车位场景下提供额外的路径规划支持，但不是安全链路的必要环节。
- **为什么参数量规划在60K左右？** OGM输出目标为160×130=20,800个栅格（覆盖车辆前后约±8m、侧向±6.5m，10cm分辨率），当前 v2 实现（U-Net 带跳跃连接）实际参数 **58,369**，float32 权重约 228 KB；引入跳跃连接后解码器输入通道从 64 增加至 96（dec1）和 48（dec2），捕捉传感器方向空间细节，参数略高于 v1 的纯上采样方案，但 OGM 侧向区域的预测精度有明显改善。INT8 量化后权重约 57 KB，满足 ECU 显存约束。

### 2.4 各算力平台推理时间估算

> **说明**：以下数据为设计阶段估算值，基于各平台官方算力规格和典型神经网络推理效率（float32 约50%利用率，INT8 约70%利用率）推算，**不代表实测结果**。实测数据须在目标硬件完成量化部署后补充。帧周期 100ms（10Hz），推理时间需 ≤ 10ms（留 ≥ 90% 控制裕量）。

#### 2.4.1 平台推理时间对比

| 算力档位 | 典型平台 | M1 ×12 | M2-SC ×6ch INT8 | M2-SA T=1 INT8 | M6 INT8 | **全链路** | **10Hz 可行性** |
|---------|---------|--------|----------------|---------------|---------|----------|--------------|
| **MCU float32**<br>< 0.5 TOPS | TC397 TriCore @300MHz<br>S32K3 Cortex-M7 @160MHz | < 0.1 ms | ⚠️ 15 ~ 30 ms | ⚠️ 1 ~ 2 ms<br>（无SIMD，含softmax exp开销） | 不推荐 | ~22 ms | ❌ 超出预算<br>（M2-SC为瓶颈） |
| **MCU INT8**<br>< 0.5 TOPS | 同上（INT8 量化） | < 0.05 ms | **4 ~ 8 ms** | ⚠️ 0.5 ~ 1 ms<br>（无SIMD，整数运算可行但效率低） | 不推荐 | **~7 ms**（含SA） | ✅ 裕量 > 82% |
| **MCU + 向量 DSP**<br>1 ~ 3 TOPS | TC3xx + AURIX AI 加速<br>ST SPC58 + DSP | < 0.05 ms | **1 ~ 3 ms** | 0.3 ~ 0.8 ms | ⚠️ 2 ~ 5 ms<br>（受内存带宽限制，2D特征图IO开销大） | ~3 ms（不含M6）<br>⚠️ 5~8 ms（含M6） | ✅ 不含M6 裕量>97%<br>⚠️ 含M6 视实现质量而定 |
| **汽车级 SoC**<br>2 ~ 5 TOPS | NXP S32G3<br>Renesas V3H / V4H | < 0.05 ms | **0.5 ~ 1.5 ms** | 0.1 ~ 0.2 ms | 3 ~ 8 ms | ~1.5 ms（不含M6） | ✅ 裕量 > 98% |
| **高算力 SoC**<br>10+ TOPS | NVIDIA DRIVE Orin<br>Qualcomm SA8295 | < 0.01 ms | **< 0.3 ms** | < 0.1 ms | < 1 ms | **< 1.5 ms** | ✅ 裕量 > 98% |

#### 2.4.2 关键结论

1. **MCU float32 部署不可行**：M2-SC 在 float32 下对6路有效通道的推理约15~30ms，远超10ms预算。**INT8 量化是 MCU 部署的前提条件，不可省略**。

2. **最低可部署配置（MCU INT8）**：仅部署 M1+M2-SC INT8，总推理约5~8ms，满足10Hz实时要求，无需额外 DSP。适合成本敏感的量产方案。

3. **推荐基准配置（MCU+DSP INT8）**：部署 M1+M2-SC+M2-SA（T=1）INT8，总推理≤3ms，裕量>97%，支持M2-SA空间注意力带来的误报率降低。

4. **M2-SA 在纯标量 MCU 上的效率评估**：M2-SA 计算量仅 58K MACs，即便在无 SIMD 的纯标量 MCU 上也可以运行（float32 约1~2ms，INT8 约0.5~1ms），不会超出帧时间预算。但 MultiheadAttention 的矩阵乘法在无向量指令的 MCU 上效率极低（利用率仅约5~10%），CPU 占用显著高于预期算力比例。**若 MCU 无向量 DSP，建议保持 `m2_sa_enabled: false`**，将 CPU 资源留给控制算法；若已部署向量 DSP，则可启用以获得空间注意力精化收益。

5. **M6 在各平台的部署门槛**：M6 v2 参数 ~58K（较 v1 增加约 60%，因 U-Net 跳跃连接扩展了解码器输入通道）、MACs ~12M，约为 M1+M2-SC+M2-SA 全链路的 2.5 倍。在 MCU+向量DSP（1~3 TOPS）上理论算力充足，但 2D-CNN 的多层特征图需要频繁读写内存，受嵌入式 DSP 内存带宽限制，实际推理约 2~5ms；加上 M1+M2-SC+M2-SA 的 ~3ms，含M6总链路为 5~8ms，理论上可满足10Hz要求。**MCU+DSP 平台上 M6 的实际可用性取决于实测结果**，需完成 INT8 量化并对 2D 卷积进行内存布局优化（如 NCHW tile化、DMA预取）后确认。5 TOPS+ SoC 上 M6 推理约 1~3ms，整体裕量充足，是 M6 的推荐部署平台。

#### 2.4.3 推荐配置矩阵

| 配置名称 | `m2_sa_enabled` | `temporal_frames` | 最低算力要求 | 推荐平台示例 |
|---------|----------------|-----------------|------------|------------|
| 基础配置（M1+M2-SC） | `false` | 1 | MCU INT8 | TC397 / S32K3 |
| 标准配置（+M2-SA 单帧） | `true` | 1 | MCU + 向量DSP | TC3xx+AURIX / SPC58 |
| 时序配置（+M2-SA 多帧） | `true` | 3 | 2~5 TOPS SoC | S32G3 / V4H |
| 全功能配置（含M6） | `true` | 5 | 5~10 TOPS SoC | DRIVE Orin / SA8295 |

#### 4.1.2 标注项目

每一条包络样本须标注以下字段：

| 字段名 | 类型 | 取值 | 说明 |
|--------|------|------|------|
| `obstacle_class` | enum | 0~8 | 主要障碍物类别（见第2章9类定义） |
| `confidence` | enum | High / Medium / Low | 标注员对本条标注的置信度评估 |
| `material_hardness` | float | 0.0~1.0，步长0.1 | 目标材质硬度估计（0=最软，1=最硬） |
| `is_suspended` | bool | true/false | 是否为悬空障碍物（类别6） |
| `suspension_height_cm` | int | 0~350，-1=未知 | 悬空净高估计（cm），仅类别6时填写 |
| `target_range_m` | float | 实测距离（m） | 障碍物到传感器的实测距离（从EDI读取） |
| `envelope_quality` | enum | Clear / Noisy / Invalid | 包络信号质量评估 |
| `annotation_notes` | string | 可选 | 特殊情况说明（如多目标重叠） |

#### 4.1.3 标注判定规则

**主类别判定优先级**（当多类特征同时出现时按优先级选取）：

1. **行人（类别2）** > 车辆（类别1）> 其他所有类：行人安全优先
2. **悬空障碍物（类别6）**：当包络出现双峰（地面回波+高位障碍物回波），且两峰时间差对应净高<3m时，标注为类别6，而不是类别0（墙体）
3. **湿滑地面（类别8）**：仅当摄像头图像确认为积水/光滑地面，且包络幅度明显高于同距离干燥地面时标注
4. **空旷/噪声（类别4）**：包络序列无明显峰值，或 EDI 置信度<50 时标注为类别4

**不允许的标注行为**：
- 不允许对置信度为 Low 的样本直接上线（须退回采集或标记为待确认）
- 不允许将 envelope_quality=Invalid 的样本标注为非类别4（无效包络只能标注为空旷/噪声）
- 不允许一条样本同时标注两个主类别（必须选择唯一主类别）

#### 4.1.4 标注工具要求

- 标注工具须提供包络波形的交互式放大/缩小功能
- 标注工具须同步播放对应时刻的摄像头视频（与包络帧精确对齐，偏差<50ms）
- 标注工具须内置类别定义说明卡（鼠标悬停显示每个类别的标准示例图）
- 标注结果须实时保存（防止丢失），支持撤销最近10步操作

### 4.2 M1：障碍物有效性标注规范

M1 的标注直接从 M2 的标注结果中**自动派生**，无需单独人工标注：

- M2 标注为类别 0、1、2、3、6、7 的样本 → M1 标签 = 1（有效，有障碍物）
- M2 标注为类别 4（空旷）或 5（地面杂波）的样本 → M1 标签 = 0（无效，无障碍物）
- M2 标注为类别 8（湿滑地面）的样本 → M1 标签 = 0（湿滑地面本身不作为障碍物）

> **注意**：M1 数据集中类别 0/1 须保持平衡，若有效样本数量超过无效样本 2 倍以上，须从无效样本中追加采集或上采样。

### 4.3 标注质量控制

#### 4.5.1 双人标注与仲裁机制

- **初标**：由初级标注员完成，速率约200帧/小时
- **复核**：由高级标注员对初标结果进行 20% 随机抽检复核，若不符合率>5%，全批次退回初标员修正
- **仲裁**：初标与复核意见不一致的样本，由算法工程师作为第三方裁决，裁决结果为最终标签

#### 4.5.2 标注一致性检验（Inter-Annotator Agreement）

每月对同一批次100条样本进行**盲标**（同一标注员隔周重标 + 不同标注员平行标注），计算 Cohen's Kappa 系数：

| Kappa 值 | 一致性评价 | 处置措施 |
|---------|---------|---------|
| ≥ 0.80 | 一致性优秀 | 正常发布 |
| 0.60~0.79 | 一致性良好 | 正常发布，记录改善建议 |
| 0.40~0.59 | 一致性中等 | 重新审查该批次标注指南，补充示例 |
| < 0.40 | 一致性差 | 暂停该类别标注，召开专项澄清会议 |

### 6.2 M1：快速特征融合 MLP 训练规范

#### 6.2.1 网络结构

```python
class ObstacleDetectionMLP(nn.Module):
    """单路 EDI 特征 → 有效性概率  (20 → 32 → 16 → 1)"""
    def __init__(self):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(20, 32),
            nn.BatchNorm1d(32),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(32, 16),
            nn.BatchNorm1d(16),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(16, 1),
            nn.Sigmoid(),  # 输出单路有效性概率标量
        )

    def forward(self, x):
        return self.layers(x).squeeze(-1)   # (B,)
```

> **推理方式说明**：同一帧的12路传感器通道共享模型权重，逐通道独立推理，每帧需调用推理函数12次，分别输入各通道的20维EDI特征，得到12个独立的有效性概率。

#### 6.2.2 损失函数

采用 **Binary Cross-Entropy with positive class weight**，每个样本为单路二分类（标量输出）：

```
Loss = -[w_pos × y × log(p̂) + (1-y) × log(1-p̂)]
```

其中 `w_pos`（正样本权重）根据训练集正负样本比例的倒数计算，默认初始值为 1.0，可随数据集统计结果调整（见训练脚本 `pos_weight` 配置项）。

#### 6.2.3 训练超参数

| 超参数 | 推荐值 | 搜索范围 |
|--------|--------|---------|
| 优化器 | Adam | — |
| 初始学习率 | 1e-3 | 1e-4 ~ 5e-3 |
| 学习率衰减 | CosineAnnealing，T_max=50epoch | — |
| Batch Size | 512 | 128 ~ 1024 |
| 最大训练轮次 | 100 epoch | — |
| Early Stopping 耐心值 | 15 epoch（监控验证集F1） | — |
| 权重正则化 | L2，系数1e-4 | 1e-5 ~ 1e-3 |

#### 6.2.4 评估指标

M1 以逐通道二分类指标衡量，每个样本代表一路传感器通道（训练集50400个样本 = 4200帧 × 12通道）：

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| 精确率（Precision） | TP / (TP+FP) | ≥ 0.92 |
| 召回率（Recall） | TP / (TP+FN) | ≥ 0.95（安全关键，优先高召回） |
| F1 Score | 2×P×R/(P+R) | ≥ 0.93 |
| 漏检率（Miss Rate） | FN/(FN+TP) | ≤ 0.05（核心指标） |
| 虚警率（False Alarm Rate） | FP/(FP+TN) | ≤ 0.08 |

### 6.3 M2-SC：单通道分类骨干训练规范（Layer-2A）

#### 6.3.1 网络结构

```python
class ObstacleClassifier1DCNN(nn.Module):
    def __init__(self, num_classes=9, use_aux_features=False):
        super().__init__()
        input_len = 258 if use_aux_features else 256
        
        self.conv_layers = nn.Sequential(
            nn.Conv1d(1, 16, kernel_size=7, padding=3),
            nn.BatchNorm1d(16), nn.ReLU(), nn.MaxPool1d(2),
            
            nn.Conv1d(16, 32, kernel_size=5, padding=2),
            nn.BatchNorm1d(32), nn.ReLU(), nn.MaxPool1d(2),
            
            nn.Conv1d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm1d(64), nn.ReLU(), nn.MaxPool1d(2),
        )
        self.global_avg_pool = nn.AdaptiveAvgPool1d(1)
        
        self.fc1 = nn.Sequential(
            nn.Linear(64, 32), nn.ReLU(), nn.Dropout(0.3)
        )
        # 主分类头：9类
        self.classifier = nn.Linear(32, num_classes)
        # 辅助回归头1：材质硬度
        self.hardness_head = nn.Sequential(nn.Linear(32, 1), nn.Sigmoid())
        # 辅助回归头2：悬空高度（仅类别6有意义）
        self.height_head = nn.Sequential(nn.Linear(32, 1), nn.ReLU())
    
    def forward(self, x, return_features=False):
        x = self.conv_layers(x)
        x = self.global_avg_pool(x).squeeze(-1)
        feat = self.fc1(x)                           # [B, 32] ← 中间特征分岔点
        if return_features:
            return feat                              # 供 M2-SA 消费，跳过分类头
        return {
            'class_logits': self.classifier(feat),   # [B, 9]
            'hardness': self.hardness_head(feat),    # [B, 1]
            'suspension_height': self.height_head(feat) # [B, 1]，单位m
        }
```

#### 6.3.2 损失函数

多任务联合损失：

```
L_total = L_cls + λ₁ × L_hardness + λ₂ × L_height

其中：
  L_cls = CrossEntropyLoss（带类别权重）
  L_hardness = MSELoss（仅对置信度 High 的样本计算）
  L_height = MSELoss（仅对类别6的样本计算）
  
  λ₁ = 0.1（材质硬度任务权重）
  λ₂ = 0.5（悬空高度任务权重）
```

| 优化器 | AdamW（weight_decay=1e-4） |

| 学习率调度 | OneCycleLR，max_lr=5e-3，total_steps=训练总steps |

### 6.4 M2-SA：组内空间注意力模块训练规范（Layer-2B）

#### 6.4.1 模块定位与设计原则

M2-SA 是 M2-SC 的**可选精化层**，在单帧模式（T=1）下执行组内空间自注意力，在多帧模式（T>1）下额外执行时序融合。两者共用同一套注意力权重，通过配置文件中的 `temporal_frames` 参数在运行时切换行为，**无需重新训练骨干**。

**降级机制**：当组内有效通道 N=1 时，自注意力退化为恒等映射，M2-SA 计算量接近零，自动回退至 M2-SC 单路输出。

**多帧配置能力**：

| `temporal_frames` | 模块行为 | 帧缓存 SRAM | 适用硬件 |
|---|---|---|---|
| 1 | 仅空间注意力，无帧缓存 | 0 KB 额外 | MCU / 2 TOPS |
| 3 | 时序加权池化 + 空间注意力 | ≈ 4.6 KB | 2~5 TOPS |
| 5 | 学习型时序注意力 + 空间注意力 | ≈ 7.7 KB | 5~10 TOPS |
| 8 | 全配置，可选 CAN 上下文 | ≈ 12.3 KB | 10 TOPS+ / GPU |

#### 6.4.2 网络结构

```python
class M2GroupAttention(nn.Module):
    """
    组内空间注意力精化模块（M2-SA / Layer-2B）

    单帧模式 (T=1)：
        输入  feat:      [B, N, 32]    N=组内有效通道数（前组或后组，≤6）
        输入  positions: [B, N] int    0~5，通道在组内的位置编号
        输出  refined:   dict          精化后 class_logits/hardness/height

    多帧模式 (T>1, 需配置 temporal_frames>1)：
        输入  feat_seq:  [B, T, N, 32] T帧历史特征序列
        内部先执行时序融合 → [B, N, 32] → 再执行空间注意力
    """
    def __init__(self, feat_dim=32, num_classes=9, num_positions=6):
        super().__init__()
        # 位置编码：0=侧向, 1=角落, 2=中心左, 3=中心右, 4=角落, 5=侧向
        self.pos_emb = nn.Embedding(num_positions, feat_dim)

        # 空间自注意力（组内 N 个通道互相参考）
        self.spatial_attn = nn.MultiheadAttention(
            embed_dim=feat_dim, num_heads=4, batch_first=True, dropout=0.1
        )
        self.spatial_norm = nn.LayerNorm(feat_dim)

        # 时序融合（仅 T>1 时激活）
        self.temporal_score = nn.Linear(feat_dim, 1)  # 每帧重要性得分

        # 输出头（与 M2-SC 损失函数共用形式）
        self.cls_head      = nn.Linear(feat_dim, num_classes)
        self.hardness_head = nn.Sequential(nn.Linear(feat_dim, 1), nn.Sigmoid())
        self.height_head   = nn.Sequential(nn.Linear(feat_dim, 1), nn.ReLU())

    def _temporal_fusion(self, feat_seq):
        """feat_seq: [B, T, N, 32] → [B, N, 32]"""
        B, T, N, D = feat_seq.shape
        scores = self.temporal_score(feat_seq).squeeze(-1)          # [B, T, N]
        time_bias = torch.linspace(0.0, 1.0, T,
                                   device=feat_seq.device).view(1, T, 1)
        weights = torch.softmax(scores + time_bias, dim=1)          # [B, T, N]
        return (feat_seq * weights.unsqueeze(-1)).sum(dim=1)        # [B, N, 32]

    def forward(self, feat, positions, padding_mask=None, feat_seq=None):
        """
        feat:         [B, N, 32]       当前帧特征（T=1 时直接使用）
        positions:    [B, N] int       通道位置编号 0~5
        padding_mask: [B, N] bool      True=padding 通道，不参与注意力
        feat_seq:     [B, T, N, 32]    多帧序列（T>1 时传入，否则为 None）
        """
        if feat_seq is not None:
            feat = self._temporal_fusion(feat_seq)   # 多帧 → 单帧压缩

        x = feat + self.pos_emb(positions)           # 叠加位置编码
        attn_out, _ = self.spatial_attn(
            x, x, x, key_padding_mask=padding_mask
        )
        x = self.spatial_norm(x + attn_out)          # 残差 + LayerNorm

        return {
            'class_logits': self.cls_head(x),        # [B, N, 9]
            'hardness':     self.hardness_head(x),   # [B, N, 1]
            'suspension_height': self.height_head(x) # [B, N, 1]
        }
```

#### 6.4.3 损失函数

M2-SA 与 M2-SC 共用损失函数形式，以精化后输出为主损失，同时保留骨干辅助监督：

```
L_total = L_cls_SA + 0.3 × L_cls_SC + 0.1 × L_hardness + 0.5 × L_height + 0.05 × L_consistency

其中：
  L_cls_SA     = CrossEntropyLoss（M2-SA 精化输出，主损失）
  L_cls_SC     = CrossEntropyLoss（M2-SC 骨干输出，辅助监督，权重 0.3）
  L_hardness   = MSELoss（仅对标注置信度 High 的样本计算）
  L_height     = MSELoss（仅对类别6的样本计算）
  L_consistency = ||feat_i - feat_j||²（同帧同类别且距离满足几何约束的通道对）
```

#### 6.4.4 训练策略

| 阶段 | 操作 | Epoch 数 | 学习率 |
|---|---|---|---|
| Stage 1 | 冻结 M2-SC 骨干，仅训练 M2-SA 全部层 | 20 | 5e-4 |
| Stage 2 | 解冻全部，联合微调 | 80 | 5e-5（线性 warmup 5 epoch） |

Stage 1 使用已训练好的 M2-SC 权重初始化骨干，仅需约 6K 新增参数（pos_emb + attention + output heads），训练成本极低。

#### 6.4.5 多帧支持与向后兼容性

M2-SA 的时序融合层（`_temporal_fusion`）参数量约 33 个（Linear(32,1) + time_bias），可单独训练或直接从零初始化。空间注意力权重在 T=1 与 T>1 时**完全相同**，无需因 `temporal_frames` 变化而重新训练骨干。

版本升级路径（与 `config.yaml` 配置对应）：

```
v1.0.0：m2_sa_enabled: false, temporal_frames: 1   → 纯 M2-SC（向后兼容）
v1.1.0：m2_sa_enabled: true,  temporal_frames: 1   → 增加空间注意力
v1.2.0：m2_sa_enabled: true,  temporal_frames: 3   → 增加时序融合（pool 模式）
v2.0.0：m2_sa_enabled: true,  temporal_frames: 5,  → 全配置（主版本变更）
        temporal_mode: "attention", use_can_context: true
```

### 6.5 M6：场景理解 2D-CNN 训练规范（可选）

M6 对应卷积场景理解网络，以多路包络矩阵（12×256）为输入，完成 OGM 栅格占用概率预测（当前实现）。

#### 6.5.1 网络结构概述（v3，当前实装版本）

采用 **编码器 + FC 几何重映射 + 转置卷积解码器** 架构（非 U-Net 跳跃连接），并输出**双通道** OGM（占据 + 自由空间）：

**输入**：`[B, 1, 12, 256]`（单通道 12×256 包络矩阵，绝对/固定参考归一化 + 对数压缩，详见下方预处理）

**编码器（sensor-range 特征提取）**：
- Conv2d(1→16, 3×3)+BN+ReLU+MaxPool2d(2×2) → `[B,16, 6,128]`
- Conv2d(16→32, 3×3)+BN+ReLU+MaxPool2d(2×2) → `[B,32, 3, 64]`
- Conv2d(32→64, 3×3)+BN+ReLU → `[B,64, 3, 64]`
- AdaptiveAvgPool2d(3, 8) → `[B,64, 3, 8]`

**FC 几何重映射（sensor-range → 笛卡尔种子栅格）**：
- Linear(64·3·8 → 256)+ReLU → Linear(256 → 16·10·8)，reshape 为种子栅格 `[B,16, 10, 8]`
- 由全连接层学习「距离-通道空间」到「车体笛卡尔空间」的几何映射，替代 U-Net 跳跃连接

**转置卷积解码器（学习式上采样）**：
- ConvTranspose2d(16→32, 4,s2,p1)+BN+ReLU → `[B,32, 20, 16]`
- ConvTranspose2d(32→24, 4,s2,p1)+BN+ReLU → `[B,24, 40, 32]`
- ConvTranspose2d(24→16, 4,s2,p1)+BN+ReLU → `[B,16, 80, 64]`
- ConvTranspose2d(16→8, 4,s2,p1)+BN+ReLU → `[B, 8,160,128]`
- Head：Conv2d(8→2, 3×3) → logits `[B,2,160,128]`
- 双线性插值至 OGM 尺寸 `[B,2,160,130]`，再 Sigmoid

**输出**：`[B, 2, 160, 130]` 双通道概率图（Sigmoid 值域 [0,1]）——`ch0=占据概率（occupancy）`，`ch1=自由空间概率（free-space）`

**OGM 网格规格**：
- 尺寸：160×130 格（行=纵向，列=横向）
- 分辨率：0.1m/格（10cm）
- 纵向范围：前后各 ±8.0m（合计 16m，160格）
- 横向范围：左右各 ±6.5m（合计 13m，130格）
- 坐标原点：车辆中心，+X 朝前，+Y 朝左

**推理预处理（必须与训练 `train_M6.normalize_envelopes()` 完全一致）**：
采用**绝对/固定参考归一化 + 对数压缩**（精度优先口径，§6.5.3）。整帧 `[12,256]` 包络先除以**固定参考** `ENV_FULL_SCALE=1.0`（ADC 满量程/标定常数，**非本帧/逐通道最大值**）并裁剪至 [0,1]，再做对数压缩：

```
x = clip(env / ENV_FULL_SCALE, 0, 1)        # ENV_FULL_SCALE = 1.0
y = log1p(K · x) / log1p(K)                  # K (LOG_COMPRESS_K) = 50.0
```

该口径保留跨帧/跨通道的**绝对幅值**（区分真实回波 vs 噪声的关键），对数压缩则将稀疏的弱回波（均值≈0.01）抬升至可分辨区间，且单调保序、对 INT8 友好。**不使用**逐通道或全局最大值归一化（二者均会抹平绝对幅值）。

**可训练参数**：775,010（float32 权重约 3.0 MB，INT8 约 757 KB）

#### 6.5.2 GT 生成方式（仿真数据）

在 PC 端仿真训练中，M6 的 OGM Ground Truth 由 `build_ogm_gt()` 自动生成双通道 `[2, 160, 130]`（ch0=占据、ch1=自由空间），无需外部标注工具：
- 对每个有效障碍物通道（类别 ∉ {Open, Clutter}），取 `ann["target_range_m"]`（即 EDI 距离）
- 结合传感器安装位置（`SENSOR_CFG[ch].x_m/y_m/yaw_deg`）计算障碍物在车辆坐标系的 XY 坐标
- ch0：在 OGM 对应格子处画 Gaussian blob（σ = `BLOB_SIGMA` = 1.0 格 = 0.10m）
- ch1：沿各传感器视锥用逆传感器模型（inverse-sensor-model）填充障碍物前的自由空间

**注意**：GT 距离来源必须使用 `ann["target_range_m"]`（EDI 实测距离）而非 `argmax(envelope)` 派生值，否则宽包络类（行人/软体）的 GT 位置将有 ±0.2m 误差。

**训练数据覆盖要求**：侧面传感器（FL-Side/FR-Side/RL-Side/RR-Side，ch0/5/6/11）须包含检测 Wall 和 Curb 的场景，否则模型将无法在 OGM 侧面区域产生热力输出。当前训练集已包含 session_train_026~031 提供此覆盖。

#### 6.5.3 精度优先 M6 变体

> **定位说明**：本节（§6.5.3）定义面向**"精度优先 + 不确定即弃权（abstain）"** 的设计：M6 不再以 OGM 占据为唯一导向，而是结合整帧点云分布进一步确认**分类/材质/高度**；对没把握、与噪声难以区分的目标**选择不报**，留待后续图像 YOLO 模型确认。其中**归一化口径（绝对/固定参考 + 对数压缩）已在 §6.5.1 当前实装版本中落地**；本节其余条目（objectness/reject 输出头、难噪声负样本、弃权率验收）为后续演进方向。

**(1) 归一化口径：改逐通道为绝对/固定参考 + 对数压缩**（**已实装**，见 §6.5.1）

精度优先目标下，跨通道**绝对幅值**是区分"真实目标 vs 噪声"以及推断材质/高度的关键判据，而逐通道/全局最大值归一化恰好将其抹平。因此当前实装版本改用：

- **绝对/固定参考归一化**：除以传感器 ADC 满量程或标定常数 `ENV_FULL_SCALE`（当前=1.0），而非本帧最大值——使绝对幅值跨帧一致，固定弃权阈值才有物理意义；
- **对数（dB）压缩**：`y = log1p(K·x)/log1p(K)`（当前 `K=50`），解决强/弱回波动态范围悬殊，INT8 下强弱回波均可表示且保序；
- **（可选）双通道输入** `[B,2,12,256]`：通道A=绝对/对数（携带幅值与 SNR），通道B=逐通道（携带纯形状），兼顾噪声弃权与弱回波形状细节（当前实装为单通道绝对/对数）。

> **归一化口径说明**：精度优先变体接受"漏掉弱的不确定目标交给 YOLO"。当前实装版本（§6.5.1）已统一采用**绝对/固定参考 + 对数压缩**，保留绝对幅值用于噪声弃权；逐通道与全局最大值归一化均不再使用。两套口径**不可混用**，须与所选目标一致。

**(2) 输出头：增加 objectness + reject 选项**

由纯 Sigmoid 占据图，扩展为 **objectness/置信度 + 类别 + 材质 + 高度 + 显式 reject**；推理时对 objectness 取高阈值，低于阈值不报（弃权），交由图像链路确认。

**(3) 训练数据与损失：难噪声负样本 + 置信度标定**

- 必须加入**难噪声负样本**与"模糊样本"，标为 background/uncertain，让模型学会在噪声上输出低置信；
- 训练后做**置信度标定**（如 temperature scaling），使弃权阈值可信、可跨场景迁移。

**(4) 验收指标：从覆盖率转向精度/弃权率**

| 指标 | 现行版本（召回优先） | 精度优先变体 |
|------|------------------|------------|
| 主指标 | OGM IoU / 栅格召回率 | 工作点 **Precision** / 虚警率 |
| 辅助指标 | 侧向区域热力覆盖 | **弃权率（abstain rate）**、标定误差（ECE） |
| 取舍 | 宁可多点亮，容忍少量误报 | 宁可不报，容忍漏报（YOLO 兜底） |

**(5) 与 M2 输出的关系：互补而非等价**

该变体保留了绝对幅值与跨通道分布，会让 M6 的信息相对 M2 **更互补、而非更接近**，二者不可相互替代：

- **结论层应一致**：对置信目标的最终类别/材质判定，M6 与 M2 应高度吻合——可将"**同位置类别吻合率**"作为一致性校验指标（不吻合通常指示数据或量化问题）。
- **信息层不等价**：M2 吃逐通道 [0,1]（仅形状、无绝对幅值、无跨通道），M6-abs 额外携带**绝对幅值/SNR、跨通道联合（三角定位/多目标）、dense 空间几何**；二者对同一属性（如材质）使用不同线索（M2 靠形状、M6 靠绝对幅值+分布），构成交叉验证。
- **退化警戒**：若将 M6-abs 设计成逐通道、单目标、per-sensor 表征，则退化为"12 路 M2 拼接"，与 M2 冗余、失去融合价值；须刻意保住上述三个增量信息。

### 9.3 交付格式规范

| 项目 | 规范 |
|------|------|
| 交付方式 | Git 仓库 Tag 发布（不得使用文件压缩包直接交付） |
| 目录结构 | 每个模型独立目录，符合第8.3节定义的目录结构 |
| 文件命名 | 全部使用英文+数字+下划线，包含版本号，禁止中文文件名 |
| 文档格式 | Markdown 格式，禁止 Word/PDF（保证 git diff 可追踪） |
| 大文件管理 | 模型权重文件使用 Git LFS 管理（onnx文件 > 10MB时） |
| 代码依赖 | 所有依赖须在 requirements.txt 中精确锁版本，确保在全新环境可一键复现 |
| 数据集版本 | 训练时使用的数据集版本须写入模型元数据，数据集本身通过数据管理平台版本控制（DVC） |

## 10. 版本管理与迭代规范

### 10.1 版本号规则

模型版本号格式：`主版本.次版本.修订号`（如 `1.2.0`）

| 变更类型 | 版本号更新 | 是否向下兼容 |
|---------|---------|-----------|
| 网络结构变更（如新增类别、层数变化） | 主版本+1，次版本和修订号归零 | **不兼容**，须同步更新SDK接口 |
| 训练数据更新（新数据 + 重新训练）/量化参数更新 | 次版本+1，修订号归零 | 兼容（接口不变） |
| Bug修复（量化错误、阈值调整）/精度小幅提升 | 修订号+1 | 兼容（透明更新） |

### 10.2 上线审批流程

```
算法工程师提交PR（含模型文件+性能报告）
           ↓
代码评审（至少1位高级工程师审阅训练代码）
           ↓
性能报告评审（验证工程师核实评估数据）
           ↓
ECU集成验证通过（集成工程师签字）
           ↓
安全工程师审核（确认安全关键指标满足）
           ↓
项目负责人批准
           ↓
Git Tag 发布 + 归档到模型仓库
```

### 10.3 模型监控与在线更新策略

模型上车后须建立以下监控机制：

| 监控项目 | 监控方式 | 告警阈值 | 处置措施 |
|---------|---------|---------|---------|
| 分类结果置信度分布 | 统计每周期主类别置信度中位数 | 连续100周期中位数<0.6 | 触发"性能下降"标志，上报诊断 |
| 新场景覆盖 | 对比在线数据与训练数据分布（特征漂移检测） | KL散度>0.3 | 触发数据补采通知，计划下一版本训练 |

**OTA 更新流程**：模型通过 OTA 升级时，新模型写入备用 Flash 分区，激活后运行5分钟观察期，若观察期内置信度中位数比旧模型下降>15%，自动回滚到旧模型。
