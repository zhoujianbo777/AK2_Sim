## 1. 背景与验证目标

仿真器现有 AI 链路（[modules/engine_ai.py](../modules/engine_ai.py)）在无 DLL 时走 PyTorch 回退：
- **M1**：逐通道 20 维 EDI 特征 → 有效性概率（阈值 0.5）。
- **M2-SC**：逐通道 256 点包络 → 9 类 + 材质硬度 + 悬空高度；M1 判无效时强制置 Open(4)。
- **M6**：整帧 12×256 包络 → 双通道 OGM（占据 + 自由空间）。

本次验证回答两个问题：
1. **真实包络喂进 M2-SC，分类是否正确？**（合成训练 vs 真实分布差异）
2. **真实 EDI 喂进 M1，有效/无效判定是否正确？**（漏检 / 虚警）

## 2. ⚠ 头号风险：包络幅值标定（必须先解决）

M2-SC 在合成包络（幅值已落在 [0,1]、峰值约 0.8）上训练。M6 预处理用 **绝对/固定参考归一化**（`ENV_FULL_SCALE=1.0` + 对数压缩，见开发说明 §6.5.3）。

因此真实探头的 ADC 原始幅值 **必须先标定到与训练同一口径**，否则：
- 幅值整体偏大/偏小 → M2 分类系统性偏移；
- M1 的 `feat[1]=amp`、`feat[4]=env.max`、`feat[8]=RMS` 等维度失真 → 有效性误判。

**标定做法（封装脚本内实现）**：
1. 记录探头 ADC 满量程 `ADC_FS`（或一次强反射墙面回波的峰值作为参考）。
2. `env_norm = clip(env_raw / CALIB_REF, 0, 1)`，其中 `CALIB_REF` 使典型墙面回波峰值落在 ~0.85（与 `env_wall` 合成峰值一致）。
3. 标定前/后各跑一次验证，对比 M2 准确率，确认标定有效（见 §6 步骤）。

> 该标定常数 `CALIB_REF` 写入 `session_meta.json` 的自定义字段 `calib_ref`，保证可追溯、可复现。

## 3. 采集方案

### 3.1 硬件与链路（Phase 0）

| 项目 | 要求 |
|------|------|
| 探头 | 单只 DJ628.30 / G32A217（按 D1），安装高度/角度记录 |
| 导出 | 每个发射周期导出：256 点包络 + EDI（距离 / 幅值 / 置信度 / 回波类型） |
| 采样率 | 10 Hz（与仿真器 100 ms/帧一致） |
| 参考 | 摄像头录像（标注用）、卷尺/激光测距（GT 距离真值） |
| 标定 | 记录 ADC 满量程或墙面参考峰值 → 求 `CALIB_REF` |

### 3.2 静态台架矩阵（Phase 1，主力）

每个「类别 × 距离 × 偏置」组合 **连续采 ≥ 30 帧（3 秒）**。

- **距离档**（卷尺真值）：0.3 / 0.5 / 1.0 / 1.5 / 2.0 / 3.0 / 4.0 m
- **侧向偏置**：0° / ±15° / ±30°（探头正前为 0°）
- **优先类别**（按 D3）：

| 类别 | 目标物示例 | 备注 |
|------|-----------|------|
| 0 Wall | 混凝土墙 / 立柱 | 高幅窄单峰基准，也用于标定 |
| 1 Vehicle | 停放车辆侧面 | 多峰 |
| 2 Pedestrian | 真人站立（**安全关键**） | 漏检率重点 |
| 7 Curb | 5/10/15 cm 路沿 | 低矮、平行与垂直两向 |
| 4 Open | 空旷无目标 | M1 负样本，验虚警 |

> 二期可补 Soft3 / Overhead6 / Wet8 / Clutter5。

### 3.4 单目标 vs FOV 内混合目标（Phase 1.5，压力测试集）

**为何先单目标**：M2-SC 架构是 **逐通道单标签**（256 点包络 → argmax 取 1 类），GT 与合成训练数据（`gen_training_data.py`）均为「每通道一个目标」。单目标采集隔离变量（幅值标定、合成→真实波形迁移），不引入标注歧义，是验证基准的正确起点，**必须先做**。

**为何仍要补混合目标**：真实探头 FOV 内目标叠加（行人贴墙、车前有路沿、悬空横梁+地面回波）很常见，且 **包络叠加正是合成单峰训练最易失效处**——这正是要暴露的弱点。

**单探头的物理限制（务必理解）**：单路 FOV 内的混合目标会 **叠加到同一条 256 点包络**，M2 只能输出一个类、M6 单通道也无法在空间上分离。因此混合目标在「单路」上验证的是：
- **M2 是否按开发说明 §4.1.3 安全优先级（行人 > 车辆 > 其他）选主类**；
- **是否漏掉了更危险的那个目标**（如把「行人+墙」判成 Wall = 危险漏检）。
- 多目标 **分离** 能力不在单路验证范围内（需多探头 / M2-SA / M6 跨通道）。

**采集与标注规则**：
- 典型组合：行人@1m + 墙@2m；路沿@0.5m + 车@1.5m；悬空横梁 + 地面回波。
- 每组合 ≥ 30 帧；按 §4.1.3 优先级取 **唯一主类** 作为 `channel_labels[chN]`。
- **不纳入主验收门控**（§6 阈值），仅作弱点探针；单列「混合目标主类命中率 / 危险漏检数」两项观察指标。

### 3.3 低速接近（Phase 2，辅助）

低速（<5 km/h）让目标由远及近连续通过，每段 ≥ 30 帧，覆盖 Wall/Vehicle/Pedestrian/Curb。同步采 CAN（车速 / 转向 / 挡位）；台架场景可用恒定值（speed=0, gear=P）。

## 4. 封装为 session（Phase 3，需新建脚本）

### 4.1 单探头 → 12 通道映射

- 把真实探头数据写入 **目标通道 chN**（默认 ch2）。
- 其余 11 路：包络全 0、EDI 全 0，GT 标 **Open(4)**。
- 仿真器对 M1<0.5 的通道强制 Open，逻辑天然吻合稀疏单路。

### 4.2 新建脚本 `tools/pack_single_probe.py`

输入：探头原始 log（建议先转成简单 CSV：每帧一行，含 timestamp、距离、幅值、置信度、回波类型 + 256 点包络）。
输出：完整 session 目录（`esi_frames.bin` / `ground_truth.json` / `session_meta.json` / `can_signals.csv`），并更新 `TestData/index.json`。

核心写帧逻辑（与 [tools/gen_training_data.py](../tools/gen_training_data.py) 完全同构，固定 13376 字节/帧）：

```python
FRAME_SIZE = 13376  # 4+4+48+48+12+12+12288+960
f_bin.write(struct.pack("<I", fid))          # frame_id  uint32
f_bin.write(struct.pack("<f", ts))           # timestamp_ms float32
f_bin.write(edi_dist.tobytes())              # float32 × 12
f_bin.write(edi_amp.tobytes())               # float32 × 12
f_bin.write(edi_conf.tobytes())              # uint8   × 12
f_bin.write(edi_echo.tobytes())              # uint8   × 12
f_bin.write(envelopes.tobytes())             # float32 × 12 × 256（已标定到[0,1]）
f_bin.write(elastic.tobytes())               # float32 × 12 × 20（全 0）
```

幅值标定在写入前完成：`envelopes[chN] = clip(env_raw / CALIB_REF, 0, 1)`。

## 6. 仿真器验证流程（Phase 4，需新建脚本）

新建 `tools/validate_single_probe.py`：加载 session，逐帧构造 `DataFrame`，调用 `AIEngine.process(frame)`，与 GT 对比并打印指标。

| 步骤 | 操作 | 通过判据 |
|------|------|---------|
| V1 | 单帧 Wall@1.0m 目检：W2 单峰、W3=Wall、W1 valid | 形状与类别正确 |
| V2 | 全 session 跑 M1：`valid_flags[chN]` vs GT 有/无障碍 | 漏检率 / 虚警率 |
| V3 | 全 session 跑 M2：`class_ids[chN]` vs GT 类别 | 准确率 / 混淆方向 |
| V4 | 标定前 vs 标定后对比 M2 准确率 | 标定确有提升 |
| V5（可选）| 目检 W4 OGM 单点占据 | 位置与 chN 几何一致 |

**验收阈值参考**（开发说明 §7.2，单探头小样本按趋势判读）：
- M1 漏检率 ≤ 5%、虚警率 ≤ 8%；
- M2 9 类准确率 ≥ 88%、行人(2)漏检率 ≤ 3%、路沿(7)漏检率 ≤ 8%。
