## 1. WSL2 环境搭建

### 1.1 启用 WSL2 并安装 Ubuntu

```powershell
# 以管理员身份在 PowerShell 中执行

# 启用 WSL 功能
wsl --install

# 查看可安装的发行版列表
wsl --list --online

# 安装 Ubuntu 22.04（推荐）
wsl --install -d Ubuntu-22.04

# 查看已安装的发行版及版本
wsl --list --verbose

# 将 WSL 默认版本设置为 2
wsl --set-default-version 2

# 将已有发行版升级到 WSL2
wsl --set-version Ubuntu-22.04 2
```

### 1.2 启动与管理 WSL

```powershell
# 启动默认 WSL 发行版
wsl

# 启动指定发行版
wsl -d Ubuntu-22.04

# 以指定用户启动
wsl -d Ubuntu-22.04 -u root

# 关闭所有 WSL 实例
wsl --shutdown

# 关闭指定发行版
wsl --terminate Ubuntu-22.04

# 导出发行版备份（镜像备份）
wsl --export Ubuntu-22.04 D:\Backup\ubuntu2204.tar

# 从备份恢复
wsl --import Ubuntu-22.04-restore D:\WSL\Ubuntu2204 D:\Backup\ubuntu2204.tar
```

### 1.3 WSL2 基础 Linux 环境初始化

> 以下命令在 WSL2 内的 Ubuntu 终端中执行。

```bash
# 更新软件源并升级系统包
sudo apt update && sudo apt upgrade -y

# 安装常用开发工具
sudo apt install -y \
    git curl wget vim build-essential \
    cmake ninja-build pkg-config \
    libssl-dev zlib1g-dev \
    python3-dev python3-pip \
    unzip zip tree htop

# 配置 Git（替换为你的信息）
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
git config --global core.autocrlf input
```

### 1.4 WSL2 与 Windows 文件互访

```bash
# 在 WSL 中访问 Windows 盘符（C盘示例）
cd /mnt/c/Users/YourName/

# 访问本项目（Windows 路径 F:\CODE\AK2_Sim）
cd /mnt/f/CODE/AK2_Sim

# 从 WSL 打开 Windows 资源管理器（当前目录）
explorer.exe .

# 在 WSL 中用 VS Code 打开当前目录
code .
```

## 2. Conda / Python 环境配置

### 2.1 安装 Miniconda（WSL2 / Linux）

```bash
# 下载 Miniconda 安装脚本（x86_64）
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda.sh

# 执行安装（-b 静默安装，-p 指定安装路径）
bash ~/miniconda.sh -b -p $HOME/miniconda3

# 初始化 conda（写入 .bashrc）
$HOME/miniconda3/bin/conda init bash

# 重新加载 shell 配置
source ~/.bashrc

# 验证安装
conda --version
```

### 2.2 安装 Miniconda（Windows PowerShell）

```powershell
# 下载安装包
Invoke-WebRequest -Uri "https://repo.anaconda.com/miniconda/Miniconda3-latest-Windows-x86_64.exe" -OutFile "$env:TEMP\miniconda.exe"

# 静默安装到 D:\miniconda3
Start-Process -FilePath "$env:TEMP\miniconda.exe" -ArgumentList "/S /D=D:\miniconda3" -Wait

# 将 conda 手动添加到当前 PowerShell 的 PATH（安装后首次使用，无需重启）
$env:PATH = "D:\miniconda3;D:\miniconda3\Scripts;D:\miniconda3\Library\bin;" + $env:PATH

# 验证 conda 是否可访问
conda --version

# 初始化 conda（写入 PowerShell profile，使每次启动自动生效）
conda init powershell

# 若提示执行策略限制，先以管理员身份执行以下命令
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 查看 PowerShell profile 路径（conda init 写入的位置）
$PROFILE

# 重新打开 PowerShell 后验证（conda init 已生效）
conda --version

# 如需将 conda 永久写入系统 PATH（所有用户、所有终端均可用）
# 以管理员身份执行：
[System.Environment]::SetEnvironmentVariable(
    "PATH",
    "D:\miniconda3;D:\miniconda3\Scripts;D:\miniconda3\Library\bin;" +
    [System.Environment]::GetEnvironmentVariable("PATH", "Machine"),
    "Machine"
)
```

### 2.3 Conda 环境管理

```bash
# 创建 AK2 项目专用环境（Python 3.10）
conda create -n ak2_sim python=3.10 -y

# 激活环境
conda activate ak2_sim

# 退出环境
conda deactivate

# 查看所有环境
conda env list

# 删除环境
conda env remove -n ak2_sim

# 导出当前环境依赖（用于迁移/共享）
conda env export > environment.yml

# 从 yml 文件重建环境
conda env create -f environment.yml

# 克隆已有环境
conda create -n ak2_sim_bak --clone ak2_sim
```

## 3. 项目依赖安装

### 3.1 安装 Python 依赖包

```powershell
# ── Windows PowerShell（推荐开发环境）──
conda activate ak2_sim
cd F:\CODE\AK2_Sim

# 从 requirements.txt 一键安装（推荐，版本已锁定）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

```bash
# ── WSL2 / Linux 下等效命令 ──
conda activate ak2_sim
cd /mnt/f/CODE/AK2_Sim
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

requirements.txt 包含的核心依赖说明：

| 包名 | 用途 | 备注 |
|------|------|------|
| `PyQt6` | GUI主框架（多窗口） | 设计说明书规定，**非 PyQt5** |
| `pyqtgraph` | 高性能实时波形绘制 | W2/W5 窗口 |
| `matplotlib` | 静态图表 | W3雷达图、W6混淆矩阵 |
| `numpy` | 数值计算 | 特征预处理、OGM运算 |
| `pyyaml` | 配置文件解析 | config.yaml |
| `onnxruntime` | AI模型CPU推理 | PC仿真阶段使用 |
| `scipy` / `tqdm` | 统计计算 / 进度条 | 批量评估模式 |
| `pyserial` | 串口通信 | HIL扩展（可选） |
| `h5py` | HDF5数据读写 | 可选 |

```powershell
# 验证关键包是否安装成功
python -c "import PyQt6; import pyqtgraph; import numpy; import yaml; import onnxruntime; print('所有核心依赖验证通过')"
```

> **注意**：`torch`（PyTorch）用于模型训练，仿真软件运行阶段不需要，
> 需要时单独安装（见第5章）。

### 3.2 CUDA / cuDNN 环境验证


## 4. 仿真软件常用操作命令

### 4.1 启动仿真主程序

```bash
conda activate ak2_sim
cd /mnt/f/CODE/AK2_Sim    # 或 Windows 下 F:\CODE\AK2_Sim

# 启动 PC 端仿真软件主界面
python main.py

# 指定配置文件启动
python main.py --config config/sim_config.yaml

# 命令行批量回放模式（不启动 GUI）
python main.py --batch --data TestData/scenario_01 --output results/
```

### 4.2 数据管理

```bash
# 列出可用测试场景
python tools/list_scenarios.py

# 检查测试数据完整性
python tools/check_data.py --path TestData/

# 格式转换（原始采集数据 → 仿真输入格式）
python tools/convert_raw.py --input RawData/ --output TestData/
```

```bash
# AI 算法 vs 传统算法双路对比
python run_compare.py --scenario TestData/scenario_01 --ai model/ak2_model.onnx

python run_compare.py --scenario TestData/ --report --output reports/compare_result.html
```

## 5. AI 模型训练与导出

### 5.1 模型训练

```bash
conda activate ak2_sim

# 启动训练（使用默认配置）
python train.py --config config/train_config.yaml

# 指定 GPU 训练
CUDA_VISIBLE_DEVICES=0 python train.py --config config/train_config.yaml

# 恢复中断的训练
python train.py --config config/train_config.yaml --resume checkpoints/latest.pth

# 查看训练日志（TensorBoard）
tensorboard --logdir runs/ --port 6006
# 浏览器访问 http://localhost:6006
```

### 5.2 模型导出（PyTorch → ONNX）

```bash
# 导出为 ONNX 格式（ECU 端部署使用）
python export_onnx.py --checkpoint checkpoints/best.pth --output model/ak2_model.onnx

# 验证 ONNX 模型
python -c "
import onnx
model = onnx.load('model/ak2_model.onnx')
onnx.checker.check_model(model)
print('ONNX model check passed')
"

# 使用 onnxruntime 推理验证
python tools/validate_onnx.py --model model/ak2_model.onnx --data TestData/sample.bin
```

### 5.3 模型量化（INT8，用于 ECU 部署）

```bash
# 静态量化（需要校准数据集）
python quantize.py \
    --model model/ak2_model.onnx \
    --calib TestData/calib/ \
    --output model/ak2_model_int8.onnx \
    --mode static

# 验证量化前后精度差异
python tools/compare_accuracy.py \
    --model-fp32 model/ak2_model.onnx \
    --model-int8 model/ak2_model_int8.onnx \
    --data TestData/scenario_01
```

## 6. 动态库编译与调试

### 6.1 编译推理动态库（Linux/WSL2）

```bash
# 进入推理库源码目录
cd /mnt/f/CODE/AK2_Sim/sdk/inference_lib

# 创建并进入构建目录
mkdir -p build && cd build

# CMake 配置（Release 版本）
cmake .. -DCMAKE_BUILD_TYPE=Release -GNinja

# 编译
ninja -j$(nproc)

# 安装到项目 lib 目录
ninja install

# 验证动态库
ldd ../lib/libak2_infer.so
```

### 6.2 编译推理动态库（Windows，MSVC）

```powershell
# 进入推理库源码目录
cd F:\CODE\AK2_Sim\sdk\inference_lib

# 创建构建目录
mkdir build; cd build

# CMake 配置（x64 Release）
cmake .. -G "Visual Studio 17 2022" -A x64 -DCMAKE_BUILD_TYPE=Release

# 编译
cmake --build . --config Release --parallel

# 验证 DLL
dumpbin /EXPORTS ..\lib\ak2_infer.dll
```

### 6.3 GDB 调试推理库（WSL2）

```bash
# 以 Debug 模式编译
cmake .. -DCMAKE_BUILD_TYPE=Debug -GNinja && ninja -j$(nproc)

# 启动 GDB 调试
gdb --args python main.py --config config/sim_config.yaml

# GDB 常用命令
# (gdb) break ak2_infer.cpp:128   # 设置断点
# (gdb) run                        # 运行
# (gdb) bt                         # 打印调用栈
# (gdb) info locals                # 查看局部变量
# (gdb) next / step                # 单步执行
# (gdb) continue                   # 继续运行
# (gdb) quit                       # 退出
```

## 8. 常见问题排查

### 8.1 WSL2 无法访问 GPU

```bash
# 检查 WSL2 版本（需 ≥ 5.10 内核）
uname -r

# 确认 Windows 侧驱动版本（需 ≥ 470.x）
# 在 Windows PowerShell 中运行：
nvidia-smi.exe

# 若 WSL2 内 nvidia-smi 仍无输出，重启 WSL
wsl --shutdown
wsl -d Ubuntu-22.04
```

### 8.2 conda activate 在 PowerShell 中失效

```powershell
# 允许 PowerShell 执行脚本
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 重新初始化 conda
conda init powershell

# 重新打开 PowerShell 窗口后验证
conda activate ak2_sim
```

### 8.3 ONNX 推理结果与 ECU 端不一致

```bash
# 检查输入数据精度是否一致（float32 vs float16）
python tools/check_precision.py --model model/ak2_model.onnx

# 检查算子版本（opset）是否与 ECU 侧推理引擎匹配
python -c "
import onnx
m = onnx.load('model/ak2_model.onnx')
print('opset version:', m.opset_import[0].version)
"

# 强制指定 opset 重新导出
python export_onnx.py --checkpoint checkpoints/best.pth \
    --output model/ak2_model_opset11.onnx --opset 11
```

### 8.4 pip 安装包时 SSL 报错

```bash
# 临时跳过证书校验（不推荐长期使用）
pip install <package> --trusted-host pypi.tuna.tsinghua.edu.cn \
    -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或更新根证书
pip install --upgrade certifi
```

### 8.5 WSL2 内存占用过大

```powershell
# 在 %USERPROFILE%\.wslconfig 中添加内存限制
# 用记事本创建或编辑该文件：
notepad "$env:USERPROFILE\.wslconfig"
```

```ini
# .wslconfig 内容示例
[wsl2]
memory=8GB       # 限制 WSL2 最大内存
processors=4     # 限制 CPU 核心数
swap=2GB
```

```powershell
# 修改后重启 WSL2 生效
wsl --shutdown
```
