## 1. 概述

传统障碍物识别算法是基于**规则的分级判决系统**，完全由 Python 实现，无机器学习模型依赖。其设计目标是在不依赖训练数据的前提下，对 12 路超声波 EDI 帧数据进行实时处理，输出每路通道的有效性标志和 5 类障碍物分类结果。

该算法代表 AK2 系统在引入 AI 能力之前的**基准感知方案**（即 AK1 方案），用于在对比模式下与 AI 算法的输出进行并排比较。

**处理粒度**：每帧 12 路通道**独立处理**，各路之间无相互依赖。

## 2. 输入数据规格

每帧输入来自 `DataFrame` 对象，算法使用以下字段：

| 字段名 | 类型 | 维度 | 说明 |
|--------|------|------|------|
| `edi_confidence` | uint8 | [12] | 12路 EDI 置信度，范围 0~100 |
| `edi_amplitude` | float32 | [12] | 12路 EDI 幅度（归一化，0~1） |
| `edi_distance` | float32 | [12] | 12路 EDI 测距结果（m） |
| `envelopes` | float32 | [12, 256] | 12路归一化包络序列（每路 256 采样点） |
| `elastic_features` | float32 | [12, 20] | 弹性波特征矩阵（保留区，当前填零） |

## 3. 输出结果规格

算法返回 `AlgoResult` 对象，各字段说明如下：

| 字段名 | 类型 | 维度 | 说明 |
|--------|------|------|------|
| `valid_flags` | float32 | [12] | 各路有效性分数（0~1，≥1.0 表示完全有效） |
| `class_ids` | uint8 | [12] | 各路主分类 ID（0~4） |
| `class_probs` | float32 | [12, 9] | 各路类别概率，前5列为传统算法输出，后4列补零与 AI 输出格式对齐 |
| `collision_probs` | float32 | [4] | 碰撞类型概率（独热编码） |
| `collision_type` | int | 标量 | 碰撞类型 ID（0~3） |
| `inference_time_ms` | float | 标量 | 本帧处理耗时（ms） |
| `engine_type` | str | 标量 | 固定为 `"Traditional"` |

**5类障碍物定义**：

| ID | 英文名 | 缩写 | 中文说明 |
|----|--------|------|---------|
| 0 | Hard | Ha | 硬质障碍物（墙体、混凝土柱等） |
| 1 | Soft | So | 软质/小型障碍物（植被、锥桶等） |
| 2 | Open | Op | 空旷/无障碍物 |
| 3 | Ground Clutter | Cl | 地面杂波/多路径干扰 |
| 4 | Unknown | ? | 无法判断 |

## 4. 算法流程总览

对每一帧，算法按以下顺序执行（12路通道循环独立处理）：

```
输入帧（EDI特征 + 包络序列）
        │
        ▼ 逐通道循环（ch = 0 ~ 11）
┌───────────────────────────────────────┐
│                                       │
│  Step 1：幅度噪底动态估计              │
│    ↓                                  │
│  Step 2：有效性判决（双门限 + 去抖动） │
│    ↓                                  │
│  Step 3：障碍物分类（5类规则树）       │
│    ↓                                  │
│  Step 4：分类置信度计算               │
│                                       │
└───────────────────────────────────────┘
        │
        ▼
结果组装 → AlgoResult
```

## 5. 第一步：幅度噪底动态估计

### 5.1 目的

由于不同传感器、不同安装位置、不同工况下的背景噪声水平差异较大，算法不使用固定噪底，而是对每路通道独立维护一个**滑动窗口动态噪底估计器**。

### 5.2 实现方式

| 参数 | 值 | 说明 |
|------|-----|------|
| 滑动窗口长度 | 30帧 | 存储最近30个"安静帧"的幅度 |
| 入队条件 | `amp < 0.15` | 仅幅度低于 0.15 的帧才认为是无障碍物的"背景帧"，避免有障碍物时的幅度污染噪底估计 |
| 默认噪底 | 0.01 | 滑动窗口为空（初始状态）时的默认值 |

```
noise_baseline = mean(amp_history[ch])   # 最近30帧安静幅度的均值
amp_threshold  = max(noise_baseline × amplitude_sigma, 0.10)
```

其中 `amplitude_sigma = 3.0`（可配置），最低幅度门限硬限制为 **0.10**，防止噪底过低导致大量误报。

### 5.3 有效幅度门限的典型取值

| 场景 | 噪底估算值 | 有效幅度门限 |
|------|-----------|------------|
| 初始（冷启动） | 0.01（默认） | 0.10（下限保护） |
| 低噪声环境 | 0.02 | max(0.02×3, 0.10) = 0.10 |
| 中等背景噪声 | 0.05 | max(0.05×3, 0.10) = 0.15 |
| 高噪声环境 | 0.08 | max(0.08×3, 0.10) = 0.24 |

---

## 6. 第二步：有效性判决（双门限 + 去抖动）

### 6.1 一级门限（单帧判断）

同时满足以下两个条件，该帧该路通道通过一级门限：

```
level1_valid = (edi_confidence ≥ 50) AND (edi_amplitude > amp_threshold)
```

| 条件 | 阈值 | 说明 |
|------|------|------|
| EDI 置信度 | `≥ 50`（可配置） | 芯片内部信号处理输出的可信度评分 |
| 幅度 | `> amp_threshold` | 自适应动态门限（见第5节） |

### 6.2 二级门限（去抖动）

单帧满足一级门限不足以判定有效，需连续多帧稳定满足：

```python
if level1_valid:
    debounce_counter[ch] += 1
else:
    debounce_counter[ch] = 0          # 任意一帧不满足则重置

is_valid = (debounce_counter[ch] >= debounce_frames)   # 默认 debounce_frames = 3
```

**防止误触发**：要求连续 **3帧**（可配置）均满足一级门限，才最终确认该通道有效。任意一帧中断则计数器清零重新计数。

### 6.3 valid_flags 的输出值

`valid_flags` 不只是二值量，而是反映去抖动进度的**连续分数**：

$$
\text{valid\_flags}[ch] = \begin{cases} 1.0 & \text{if } \text{debounce\_counter}[ch] \geq N_{debounce} \\ \dfrac{\text{debounce\_counter}[ch]}{N_{debounce}} & \text{otherwise} \end{cases}
$$

例：`debounce_frames = 3`，连续1帧满足 → `valid_flags = 0.333`，连续2帧 → `0.667`，连续3帧 → `1.0`。

## 7. 第三步：障碍物分类（5类规则树）

对通过二级门限的通道，使用包络特征进行分类。分类规则为**优先级从高到低的条件树**：

```
is_valid = False  OR  amp ≤ noise_baseline
    └──→  CLS_OPEN（2，空旷）
            ↓（否则计算包络特征）
envelope_tail = sum(envelope[20:] > 0.1)   # 第20点后幅度超过0.1的采样点数量
peak_idx = argmax(envelope)                 # 包络主峰位置

            ↓
envelope_tail > 20  AND  dist < 0.3m
    └──→  CLS_GROUND（3，地面杂波）
            ↓
amp > 0.7  AND  peak_idx < 200
    └──→  CLS_HARD（0，硬质障碍物）
            ↓
0.3 < amp ≤ 0.7
    └──→  CLS_SOFT（1，软质障碍物）
            ↓
其他
    └──→  CLS_UNKNOWN（4，未知）
```

### 7.1 各类别判别依据

| 类别 | 判别条件 | 物理含义 |
|------|---------|---------|
| Open（空旷） | 通道无效，或幅度未超过噪底 | 探测范围内无障碍物 |
| Ground Clutter（地面杂波） | 包络尾部长（>20点）且近距离（<0.3m） | 地面多路径反射形成的长尾拖尾，位置固定且近 |
| Hard（硬质） | 幅度高（>0.7）且峰值位于前段（<200采样点） | 高幅窄单峰，前沿锐利，典型于墙体/混凝土 |
| Soft（软质） | 幅度中等（0.3~0.7） | 幅度低于硬质目标，包络较宽泛，典型于植被/锥桶 |
| Unknown | 不满足以上任何规则 | 信号异常或边界情况 |

### 7.2 包络特征提取说明

| 特征 | 计算方式 | 用途 |
|------|---------|------|
| `envelope_tail` | `sum(envelope[20:] > 0.1)` | 第20个采样点之后仍超过0.1的点数，越大说明回波拖尾越长，地面杂波特征 |
| `peak_idx` | `argmax(envelope)` | 包络最大值位置，越靠前（采样点索引越小）说明障碍物越近或回波上升越陡 |

> **注意**：传统算法**不使用包络整体形状**（如峰宽、双峰结构、偏度等），仅使用幅度标量和两个简单的包络派生统计量。这是与 AI 算法（使用完整 256 点包络序列）的核心差距所在。

## 8. 第四步：分类置信度计算

置信度反映算法对分类结果的确定程度，按分类结果分三种情形计算：

### 8.1 Open 类（空旷）

```python
primary_conf = clip(1.0 - amp / amp_threshold, 0.30, 0.98)
```

幅度越远低于门限，置信度越高（最高 0.98）；幅度恰好在门限附近时置信度接近下限 0.30。

### 8.2 Unknown 类

```python
primary_conf = 0.45   # 固定低置信度
```

固定为较低值，反映算法对该结果本身就不确定。

### 8.3 有障碍物类（Hard / Soft / Ground Clutter）

$$
\text{SNR\_ratio} = \frac{\text{amp}}{\text{amp\_threshold}}
$$

$$
\text{conf\_snr} = \text{clip}\left(0.50 + 0.47 \times \left(1 - \frac{1}{\text{SNR\_ratio}}\right),\ 0.50,\ 0.97\right)
$$

$$
\text{debounce\_ratio} = \min\left(\frac{\text{debounce\_counter}}{N_{debounce}},\ 1.0\right)
$$

$$
\text{primary\_conf} = \text{conf\_snr} \times (0.6 + 0.4 \times \text{debounce\_ratio})
$$

- **SNR 对数曲线**：SNR=1（刚过门限）时置信度为 0.50，SNR→∞ 时趋近 0.97
- **去抖动加权**：去抖动尚未完成时（ramp-up 阶段），置信度乘以 0.6~1.0 的权重，反映初始几帧尚不稳定

### 8.4 概率向量生成

主分类概率确定后，将剩余概率均分给其余 4 类，再做归一化：

```python
probs[cls_id] = primary_conf
probs[其他] = (1.0 - primary_conf) / 4
probs = clip(probs, 0, 1)
probs /= probs.sum()   # 归一化确保总和为1
```

## 10. 可配置参数说明

所有阈值均通过 `config.yaml` 的 `traditional.*` 节点配置，**无需修改代码**，重启软件后生效：

| 参数路径 | 默认值 | 说明 | 调整建议 |
|---------|--------|------|---------|
| `traditional.confidence_threshold` | `50` | EDI 置信度一级门限（0~100） | 降低可减少漏检但增加误报；升高反之 |
| `traditional.amplitude_sigma` | `3.0` | 有效幅度门限 = 噪底 × 此值 | 高噪声环境可适当调大（如 4.0）降低误报 |
| `traditional.debounce_frames` | `3` | 二级去抖动所需连续帧数 | 增大可提高稳定性但增加响应延迟；减小反之 |
| `traditional.elastic_rms_threshold` | `0.5` | 弹性波碰撞 RMS 门限（保留字段） | 当前弹性波数据为零，该参数实际不生效 |

### 调参影响矩阵

| 参数 | 增大效果 | 减小效果 |
|------|---------|---------|
| `confidence_threshold` ↑ | 漏检率 ↑，误报率 ↓ | 漏检率 ↓，误报率 ↑ |
| `amplitude_sigma` ↑ | 门限更高，误报率 ↓ | 门限更低，灵敏度 ↑ |
| `debounce_frames` ↑ | 响应延迟 ↑，稳定性 ↑ | 响应延迟 ↓，抖动 ↑ |

## 11. 算法局限性

传统算法使用的特征（幅度标量、包络尾部长度、峰值位置）信息量极为有限，以下场景存在已知的系统性缺陷：

| 局限场景 | 失效原因 | AI 算法的改进方式 |
|---------|---------|----------------|
| 悬空障碍物（如限高横梁） | 仅看幅度和峰值位置，无法区分双峰结构 | M2 使用完整 256 点包络，可识别地面反射+高位反射的双峰特征 |
| 低矮路沿/路牙 | 幅度低，可能被判为 Soft 或 Unknown，不稳定 | M2 专有类别 Curb（类别7），通过包络形态精确识别 |
| 湿滑/冰面地面 | 镜面反射导致幅度异常高，被误分为 Hard | M2 专有类别 Wet（类别8），通过二次回波鬼影特征识别 |
| 行人识别 | 行人幅度中等，经常与软质障碍物混淆 | M2 专有类别 Pedestrian（类别2），置信度和召回率均高于规则方案 |
| 雨天/高湿度 | 背景幅度抬升导致噪底估计偏高，门限漂移，虚警增加 | M1 通过学习正常模式，对环境扰动更鲁棒 |
| 无材质信息 | 规则树无法推断材质硬度 | M2 输出 `material_hardness` 连续评分（0~1） |
| 无悬空高度信息 | 规则树无净高估算能力 | M2 输出 `suspension_height_m` 连续回归值 |
